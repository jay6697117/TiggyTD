# 敌人数据库

> **状态**: 设计中
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 支柱三（纪元对抗叙事）、支柱二（Build 进化自由）

---

## Overview（概述）

敌人数据库是 TiggyTD 所有史前生物单位的属性定义中心。它以静态数据表存储10种普通敌人和3个Boss的全部基础属性——包括HP、护甲率、移动速度、攻击力、特殊能力标签和所属阵营章节。敌人按三个章节阵营组织：第1章恐龙纪元、第2章冰河时期、第3章远古海洋，每个阵营有独立的视觉主题和策略特征。所有依赖敌人属性的系统（敌人AI、波次系统、伤害计算、Boss系统）均从本数据库读取，不允许在其他系统中硬编码敌人数值。

---

## Player Fantasy（玩家幻想）

**玩家幻想**：「我看穿了这波敌人的意图，我的 Build 专门为它们准备的。」

每章的史前阵营都应该给玩家传递一个清晰的「威胁信号」：恐龙纪元让玩家感受到速度与蛮力的冲击；冰河时期带来持久与寒冷的压迫感；远古海洋制造神秘与不可预测的恐惧。

玩家在看到新敌人的第一眼，应该能猜到它的威胁类型——翼龙在飞所以地面陷阱没用，猛犸象很壮所以需要穿透，菊石群数量多所以需要AOE。这种「直觉 → 验证 → 调整Build」的循环是策略满足感的来源。

Boss 战是最高光时刻：玩家应该感受到「我特意为你准备了这套Build」的针对性爽感，以及「纪元审判」打破沧龙护盾时的史诗级视觉冲击。

## Detailed Design（详细设计）

### Core Rules（核心规则）

敌人数据库以静态数据表存储，运行时只读。所有系统通过敌人ID查询属性，不得直接修改数据库条目。

#### 属性字段说明

| 字段 | 单位 | 说明 |
|------|------|------|
| `hp` | 数值 | 基础生命值 |
| `armor` | % | 护甲率，范围 0.0~0.75，输入伤害计算系统 |
| `speed` | 格/秒 | 沿路径移动速度 |
| `atk` | 数值 | 对动物塔的攻击力 |
| `ability` | 标签 | 特殊能力ID，由敌人AI系统读取执行 |
| `chapter` | 1/2/3 | 所属章节阵营 |
| `type` | normal/elite/boss | 单位类型 |
| `reward` | 金币 | 击败后掉落金币 |

#### Boss 阶段机制

**霸王龙王**
- 阶段1（HP<50%）：狂暴，speed×1.5，atk×1.3
- 阶段2（HP<25%）：召唤2只迅猛龙援军
- 弱点：被孔雀魅惑后5秒内受到伤害+50%

**猛犸象将军**
- 阶段1（HP<60%）：冲锋（直线贯穿所有单位，造成atk×150%伤害）
- 阶段2（HP<40%）：冰原风暴（全场动物塔攻速-30%，持续10秒）
- 弱点：被大象冲撞推开后3秒内受到伤害+80%

**沧龙霸主**
- 阶段1（HP<70%）：水域传送（每15秒随机传送至地图另一水域）
- 阶段2（HP<50%）：深海之怒（召唤菊石群×3组）
- 阶段3（HP<20%）：无敌护盾（持续5秒，需用「纪元审判」大招打破）
- 弱点：离开水域时受到伤害+100%

### States and Transitions（状态与转换）

数据库为静态只读结构，无状态转换。运行时流程：

1. 游戏启动时从配置文件加载全部敌人数据到内存
2. 各系统通过 `EnemyDB.get(enemy_id)` 查询，返回只读数据对象
3. 敌人AI系统在运行时读取 `ability` 标签执行特殊行为逻辑
4. Boss阶段切换由Boss系统监听HP阈值触发，不修改数据库

### Interactions with Other Systems（与其他系统的交互）

| 系统 | 交互方式 |
|------|----------|
| 伤害计算系统 | 查询 `armor` 作为护甲率输入 |
| 敌人AI系统 | 查询 `speed`、`atk`、`ability` 执行移动和攻击行为 |
| 波次系统 | 查询 `chapter`、`type` 决定每波生成哪些敌人 |
| Boss系统 | 查询Boss的HP阈值、阶段机制、弱点标签 |
| 资源/货币系统 | 查询 `reward` 计算击败敌人后的金币掉落 |
| 游戏UI系统 | 查询敌人名称、HP用于血条显示 |

## Formulas（公式）

敌人数据库本身不包含计算公式，只提供原始数值。以下是下游系统使用本数据库数值的关键公式引用：

**敌人受到伤害（来自伤害计算系统）**
```
最终伤害 = 基础伤害 × max(0.25, 1 - EnemyDB.get(id).armor × (1 - 穿透率)) × 其他修正
```

**流血持续伤害（剑齿虎）**
```
流血伤害/秒 = EnemyDB.get("saber_tooth").atk × 0.3
流血持续时间 = 3秒
总流血伤害 = 流血伤害/秒 × 3
```

**Boss狂暴阶段属性**
```
// 霸王龙王 HP<50% 后
实际speed = EnemyDB.get("trex_king").speed × 1.5
实际atk = EnemyDB.get("trex_king").atk × 1.3
```

**菊石群金币奖励**
```
总奖励 = EnemyDB.get("ammonite_swarm").reward × 存活击杀数
// 5只全灭 = 4 × 5 = 20金币
```

**沧龙弱点伤害**
```
// 沧龙霸主离开水域时
弱点修正 = 2.0（叠加伤害计算系统的弱点修正）
// 注意：此弱点与「纪元审判」弱点标记不叠加，取较大值
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 翼龙被变色龙陷阱覆盖区域经过 | 陷阱不触发（fly能力免疫地面陷阱） |
| 三角龙冲撞推开塔时目标格已有其他塔 | 被推开的塔移动到最近的空格；若无空格则冲撞无效（塔不被摧毁） |
| 菊石群5只中有1只被消灭 | 剩余4只继续独立行动；每只死亡各自结算reward=4金币 |
| 沧龙霸主传送时目标水域不存在（关卡无多个水域） | 传送能力不触发，正常移动 |
| Boss阶段2召唤援军时场上敌人已达上限 | 延迟召唤，等待场上敌人数量降至上限以下再触发 |
| 猛犸象将军冰原风暴与百兽怒吼同时激活 | 两者叠加计算：攻速先×(1+0.5)再×(1-0.3)，净结果≈+5% |
| Boss无敌护盾期间受到「纪元审判」 | 护盾立即破除，该次纪元审判伤害正常结算 |
| 沧龙弱点与纪元审判弱点标记同时存在 | 取较大值（2.0），不叠加为4.0 |
| 查询不存在的enemy_id | 返回null，调用方负责处理；记录警告日志，不崩溃 |
| 腕龙践踏伤害命中已死亡的动物塔 | 丢弃该次践踏伤害事件 |

## Dependencies（依赖关系）

### 上游依赖

无。敌人数据库是 Layer 0 系统，不依赖任何其他游戏系统。数据来源为设计时编写的静态配置文件。

### 下游依赖（这些系统依赖本数据库）

| 系统 | 使用字段 | 状态 |
|------|---------|------|
| 伤害计算系统 | `armor` | ✅ 已完成 |
| 敌人AI系统 | `speed`、`atk`、`ability` | ⬜ 未设计 |
| 波次系统 | `chapter`、`type`、`hp` | ⬜ 未设计 |
| Boss系统 | 全部Boss字段、阶段机制、弱点 | ⬜ 未设计 |
| 资源/货币系统 | `reward` | ⬜ 未设计 |
| 游戏UI系统 | 名称、`hp` 用于血条显示 | ⬜ 未设计 |

### 数据接口约定

```
EnemyDB.get(enemy_id: String) -> EnemyData
EnemyDB.get_all() -> Array[EnemyData]
EnemyDB.get_by_chapter(chapter: int) -> Array[EnemyData]
EnemyDB.get_bosses() -> Array[EnemyData]

EnemyData {
  id: String           // 唯一标识，如 "raptor"、"trex_king"
  name_key: String     // 本地化键名，如 "enemy.raptor.name"
  chapter: int         // 1 | 2 | 3
  type: String         // "normal" | "elite" | "boss"
  hp: int
  armor: float         // 0.0 ~ 0.75
  speed: float
  atk: int
  reward: int          // 击败金币奖励
  ability: String      // 特殊能力ID，如 "dash"、"fly"、"swarm"
  weakness: String     // 弱点触发条件ID，Boss专用
  phases: Array        // Boss阶段配置，普通敌人为空数组
}
```

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 迅猛龙冲刺速度倍数 | 2.0x | 1.5~2.5x | 低于1.5x冲刺感不明显；高于2.5x玩家反应不过来 |
| 三角龙护甲率 | 30% | 20%~45% | 影响穿透Build的必要性，低于20%普通攻击够用 |
| 冰河巨熊冰甲触发概率 | 20% | 10%~35% | 高于35%体验像在打随机数，低于10%能力感不强 |
| 腕龙精英HP | 800 | 600~1200 | 影响精英单位的「关卡事件感」，太低不够威胁 |
| 霸王龙王总HP | 3000 | 2000~5000 | 影响第1章Boss战时长，目标约3~5分钟 |
| 猛犸象将军冰原风暴持续时间 | 10秒 | 6~15秒 | 超过15秒玩家体验过于压抑 |
| 沧龙霸主无敌护盾持续时间 | 5秒 | 3~8秒 | 影响「纪元审判」时机决策的紧迫感 |
| 沧龙霸主水域传送间隔 | 15秒 | 10~20秒 | 低于10秒玩家来不及调整布阵 |
| Boss阶段奖励金币 | 200/350/500 | ×0.7~×1.5 | 影响玩家通关后的Build雪球空间 |
| 菊石群单只HP | 50 | 30~80 | 太高AOE需求过强，太低失去群体压迫感 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 所有13个敌人（10普通+3Boss）数据可正常加载 | 游戏启动后调用 `EnemyDB.get_all()`，确认返回13条完整记录 |
| AC-2 | 所有敌人 `armor` 在 0.0~0.75 范围内 | 遍历全部记录，断言 `0.0 <= armor <= 0.75` |
| AC-3 | 翼龙不触发地面陷阱 | 变色龙陷阱放置于翼龙路径，翼龙经过后陷阱不触发，HP不变 |
| AC-4 | 三角龙冲撞正确推开塔1格 | 三角龙攻击前排塔，确认塔被推开1格；目标格有塔时冲撞无效 |
| AC-5 | 菊石群5只各自独立寻路 | 放置菊石群，确认5个独立单位各自朝不同目标移动 |
| AC-6 | 霸王龙王HP<50%时正确触发狂暴 | Boss HP降至49%，确认speed和atk数值变化符合×1.5和×1.3 |
| AC-7 | 沧龙霸主无敌护盾被「纪元审判」打破 | 触发无敌护盾期间使用纪元审判，护盾立即消失，伤害正常结算 |
| AC-8 | 沧龙弱点与纪元审判弱点不叠加，取较大值 | 沧龙离开水域同时被纪元审判标记，确认弱点修正=2.0而非4.0 |
| AC-9 | 击败敌人正确掉落金币 | 击败迅猛龙确认+8金币；击败霸王龙王确认+200金币 |
| AC-10 | 按章节查询返回正确敌人 | `EnemyDB.get_by_chapter(1)` 返回4种恐龙纪元敌人 |
