## Overview（概述）

道具数据库是 TiggyTD 所有可装备道具的属性定义中心。它存储40种道具（25种基础道具 + 15种合成道具）的完整属性，包括效果类型、数值、合成配方和商店价格。每只动物可同时装备4件道具（无分类限制），道具效果通过伤害计算系统、状态效果系统等下游系统生效。合成系统依赖本数据库的配方定义，商店系统依赖本数据库的价格和随机池配置。数据库本身只负责存储，不包含任何逻辑。

---

## Player Fantasy（玩家幻想）

**玩家幻想**：「差一件就能合成了——再撑一波！」

道具系统的核心钩子是「合成欲望」：玩家看到手里有一件毒牙，商店里又出现了利爪，他会立刻意识到「再买一件就能合成死亡之牙」，这种「差一步」的张力是推动玩家继续的核心动力。

4件装备槽带来的是「我的这只犀牛是独一无二的配置」的个人化满足感。每件道具的选择都是一次小决策，四件道具组合在一起就是一个完整的 Build 表达。

合成道具的效果要明显强于单件基础道具，让玩家感受到「合成是值得的投资」，而不是「合成只是数值微调」。

## Detailed Design（详细设计）

### Core Rules（核心规则）

道具数据库以静态数据表存储，运行时只读。每件道具有唯一ID，合成道具通过配方字段引用原材料ID。

#### 属性字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 唯一标识，如 `sharp_claw` |
| `name_key` | String | 本地化键名 |
| `tier` | 1/2 | 1=基础道具，2=合成道具 |
| `price` | int | 商店购买价格（tier2道具不在商店出售） |
| `recipe` | Array | 合成配方，包含2个原材料ID（tier1为空） |
| `effects` | Array | 效果列表，每个效果含type、value字段 |
| `shop_weight` | int | 商店随机池权重，值越高越容易出现（0=不出现在商店） |

### States and Transitions（状态与转换）

数据库为静态只读结构，无状态转换。运行时流程：

1. 游戏启动时从配置文件加载全部道具数据到内存
2. 商店系统按 `shop_weight` 构建随机池，tier2道具不进入随机池
3. 道具装备系统通过 `ItemDB.get(item_id)` 读取效果列表并应用
4. 合成系统通过 `recipe` 字段检测是否满足合成条件

### Interactions with Other Systems（与其他系统的交互）

| 系统 | 交互方式 |
|------|----------|
| 伤害计算系统 | 读取道具的 `base_atk附加`、`穿透率`、`暴击率`、`暴击倍率` 效果值 |
| 道具装备系统 | 读取道具效果列表，应用至动物塔属性 |
| 道具合成系统 | 读取 `recipe` 字段检测合成条件，读取合成结果道具ID |
| 商店系统 | 读取 `price`、`shop_weight` 构建商店随机池 |
| 状态效果系统 | 读取中毒、减速、瘟疫等效果参数 |
| 资源/货币系统 | 读取 `gold_tooth`/`gold_empire` 的金币加成效果 |

## Formulas（公式）

道具数据库本身不包含计算公式，只提供原始数值。以下是下游系统使用本数据库数值的关键公式引用：

**道具附加伤害输入（来自伤害计算系统）**
```
基础伤害 = 动物塔基础攻击力 × (1 + 技能树攻击加成) + Σ(装备道具的base_atk附加)
// 例：装备利爪+死亡之牙 → 附加伤害 = 10 + 15 = 25
```

**穿透率叠加**
```
总穿透率 = min(1.0, Σ(装备道具的穿透率加成) + 技能树穿透率)
// 例：穿刺角(+15%) + 灵魂穿刺(+40%) = 55%，未超上限
```

**暴击率叠加**
```
总暴击率 = min(0.75, 基础暴击率 + Σ(道具暴击率加成) + 技能树暴击率加成)
```

**合成价值计算（用于UI提示）**
```
合成节省金额 = Σ(原材料价格) - 合成道具参考价值
// 例：死亡之牙 = 利爪(40) + 毒牙(50) = 90金币的材料，合成道具参考价值120
// 节省 = 90 - 120 = -30（即合成增值30金币）
```

**战争光环AOE范围**
```
覆盖格数 = π × 范围半径² ≈ π × 3² ≈ 28格
// 战争光环范围3格，约覆盖28个格子内的友军
```

**黄金帝国被动金币**
```
每局被动收入 = 3金币/10秒 × 关卡时长（秒）
// 10分钟关卡约产生180金币被动收入
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 同一动物装备2件相同基础道具 | 效果叠加（不自动合成），UI显示合成提示图标；玩家手动触发合成 |
| 合成后道具数量超过4件槽位 | 合成前检查：若合成后该塔道具总数≤4件才允许合成；否则提示「槽位已满」 |
| 灵魂链接相邻动物死亡 | 共享HP池解除，剩余HP归属存活动物；存活动物HP不低于1 |
| 灵魂链接目标移动后不再相邻 | 链接立即断开，两者HP独立，各保留断开时的HP比例 |
| 野性本能/原始本能临时buff跨波持续 | 临时buff在下一波开始时激活，该波结束时移除；不累计到下下波 |
| 咆哮石AOE攻击命中飞行单位（翼龙） | 翼龙在AOE范围内时正常受到AOE伤害（AOE非地面陷阱，不受fly能力免疫） |
| 嘲讽宝石与要塞印记同时装备 | 效果叠加：嘲讽范围3格，标记目标受伤+25%，两个条件都满足时强制攻击本塔 |
| 黄金帝国被动金币在关卡失败时 | 被动金币不计入跨局成长，仅在本局有效 |
| 碎甲石护甲降低叠加超过75% | 敌人护甲率下限为0%（不能出现负护甲），碎甲效果截断 |
| 狙击镜装备后range超过地图边界 | 攻击范围截断至地图边界，不会攻击地图外区域 |

## Dependencies（依赖关系）

### 上游依赖

无。道具数据库是 Layer 0 系统，不依赖任何其他游戏系统。数据来源为设计时编写的静态配置文件。

### 下游依赖（这些系统依赖本数据库）

| 系统 | 使用字段 | 状态 |
|------|---------|------|
| 伤害计算系统 | `effects`中的base_atk附加、穿透率、暴击率、暴击倍率 | ✅ 已完成 |
| 道具装备系统 | `effects`、`tier`、`price` | ⬜ 未设计 |
| 道具合成系统 | `recipe`、`tier` | ⬜ 未设计 |
| 商店系统 | `price`、`shop_weight`、`tier` | ⬜ 未设计 |
| 状态效果系统 | 中毒、减速、瘟疫效果参数 | ⬜ 未设计 |
| 资源/货币系统 | `gold_tooth`/`gold_empire`金币加成参数 | ⬜ 未设计 |
| 游戏UI系统 | 名称、效果描述、合成配方用于面板展示 | ⬜ 未设计 |

### 数据接口约定

```
ItemDB.get(item_id: String) -> ItemData
ItemDB.get_all() -> Array[ItemData]
ItemDB.get_by_tier(tier: int) -> Array[ItemData]
ItemDB.get_shop_pool() -> Array[ItemData]  // 返回shop_weight>0的所有tier1道具

ItemData {
  id: String
  name_key: String
  tier: int              // 1 | 2
  price: int             // tier2为0（不可购买）
  shop_weight: int       // 0=不出现在商店
  recipe: Array[String]  // tier1为空，tier2为[材料ID, 材料ID]
  effects: Array[Effect]
}

Effect {
  type: String   // "atk_bonus"|"atk_speed"|"crit_rate"|"crit_mult"|"penetration"
                 // |"hp_bonus"|"armor"|"reflect"|"regen"|"aura"|"slow"
                 // |"gold_bonus"|"range"|"move_speed"|"cooldown_reduction"
                 // |"poison"|"chain"|"aoe"|"armor_break"|"taunt"|"echo"
                 // |"soul_link"|"wild_buff"
  value: float   // 效果数值
  duration: float  // 持续时间（秒），永久效果为-1
}
```

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 基础道具平均价格 | 47金币 | 35~65 | 影响每波间隔能购买几件道具，决定Build雪球速度 |
| 合成道具增值幅度 | +30~50金币 | +20~+70 | 低于20合成动力不足；高于70合成成为唯一正确选择 |
| 每只动物装备槽数 | 4 | 3~5 | 影响单塔Build复杂度和管理负担 |
| 咆哮石AOE半径 | 1格 | 0.8~1.5 | 高于1.5与战争光环功能重叠 |
| 战争光环范围 | 3格 | 2~4 | 高于4格几乎覆盖全场，削弱布阵策略意义 |
| 灵魂链接HP共享范围 | 相邻1格 | 1~2格 | 高于2格可远程组队，削弱位置决策 |
| 黄金帝国被动金币 | 3金币/10秒 | 2~5 | 高于5金币/10秒经济雪球过快 |
| 商店基础道具最大同时出现数 | 由商店系统定义 | — | 本数据库通过shop_weight控制概率分布 |
| 碎甲石护甲降低量 | 10%/层 | 5%~15% | 高于15%2层即达上限，削弱叠加Build深度 |
| 连锁爪弹射概率 | 20% | 15%~35% | 高于35%与风暴连击差异缩小 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 所有40种道具数据可正常加载，无缺失字段 | 调用 `ItemDB.get_all()`，确认返回40条完整记录 |
| AC-2 | 25种基础道具全部出现在商店随机池中 | 调用 `ItemDB.get_shop_pool()`，确认返回25条记录且shop_weight>0 |
| AC-3 | 15种合成道具不出现在商店 | 确认所有tier2道具的shop_weight=0 |
| AC-4 | 每只动物最多装备4件道具，第5件无法装备 | 尝试给已装备4件道具的动物再装备1件，确认被拒绝并提示「槽位已满」 |
| AC-5 | 2件相同基础道具触发合成提示 | 给同一动物装备2件利爪，确认UI显示合成提示图标 |
| AC-6 | 合成后原材料消耗，产出合成道具 | 触发利爪+毒牙合成，确认两件原材料消失，死亡之牙出现在装备槽 |
| AC-7 | 穿透率叠加不超过1.0 | 装备穿刺角(+15%)+灵魂穿刺(+40%)+技能树(+60%)，确认总穿透率=1.0而非1.15 |
| AC-8 | 黄金帝国每10秒正确产生3金币 | 装备黄金帝国后等待30秒，确认金币增加9 |
| AC-9 | 碎甲石叠加不使敌人护甲率低于0% | 对护甲率20%的敌人叠加3层碎甲石（-30%），确认有效护甲率=0%而非-10% |
| AC-10 | 狙击镜range加成正确叠加 | 鹰（range=7）装备远望镜（+1.5）+狙击镜（+3），确认总range=11.5 |
