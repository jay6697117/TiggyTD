# 波次系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 游戏流程驱动（控制战斗节奏和压力曲线）

---

## Overview（概述）

波次系统控制史前生物的出场节奏：何时生成哪些敌人、生成多少、生成间隔，以及波次之间的休息时间。MVP阶段共10波，第10波为Boss波。系统负责波次倒计时、敌人生成、波次清场判定（所有敌人死亡或到达基地），以及触发波次补贴发放。波次配置为静态数据表，不在运行时随机生成（随机性系统仅提供敌人种类的轻微随机变化）。

---

## Player Fantasy（玩家幻想）

波次是游戏的呼吸节奏。战斗时的紧张感、间隔时的建造喘息、Boss波前的准备压力——这条节奏线是玩家情绪体验的主轴。第1波要让玩家感到「还好应付」，第5波开始感到「开始有压力了」，第9波结束时「撑过来了但很险」，第10波Boss出场时「决战感」爆棚。每波结束的「清场感」（最后一只敌人死亡）和补贴到账是小满足感的来源。

## Detailed Design（详细设计）

### 波次流程

```
游戏开始
  → 显示波次信息（本波敌人预览）
  → 倒计时 prepare_time 秒
  → 发放波次补贴（资源/货币系统）
  → 开始生成敌人
    → 按spawn_list逐条生成（间隔spawn_interval秒）
  → 等待清场（所有敌人死亡或到达基地）
  → 波次结束
  → 若为最终波 → 触发胜负判定
  → 否则 → 下一波倒计时
```

### 波次配置表（MVP 10波）

| 波次 | 敌人组合 | 总数 | 特殊说明 |
|------|---------|------|----------|
| 1 | 迅猛龙×5 | 5 | 教学波，单一敌人 |
| 2 | 迅猛龙×4、翼龙×3 | 7 | 引入飞行单位 |
| 3 | 三角龙×3、迅猛龙×4 | 7 | 引入charge机制 |
| 4 | 腕龙×1、迅猛龙×6 | 7 | 引入精英单位 |
| 5 | 剑齿虎×4、三角龙×2 | 6 | 章节2开始，引入bleed |
| 6 | 猛犸象×3、剑齿虎×3 | 6 | 高护甲压力 |
| 7 | 冰河巨熊×3、猛犸象×2、剑齿虎×2 | 7 | 混合压力 |
| 8 | 邓氏鱼×3、沧龙×4 | 7 | 章节3开始，引入水域机制 |
| 9 | 菊石群×2组(×5)、邓氏鱼×3 | 13 | 群体压力，需AOE |
| 10 | 霸王龙王×1（Boss） | 1 | Boss波，无补贴 |

### 生成参数

```
prepare_time = 15秒（波次间准备时间，第1波=30秒）
spawn_interval = 1.5秒（同波次内相邻敌人生成间隔）
spawn_point = 关卡定义的生成点坐标
菊石群生成：5只同时生成在spawn_point附近的5个相邻格子
```

### 清场判定

```
波次结束条件：spawn_list中所有敌人均已「离场」
离场定义：
  - 敌人HP归零（被击杀）→ 发放reward
  - 敌人到达基地（漏过）→ 通知胜负判定系统扣基地HP
两种情况均计入清场计数
全部离场 → 波次结束
```

### Boss波特殊规则

```
Boss波（第10波）：
  - 不发放波次补贴
  - Boss死亡 → 直接触发「胜利」判定
  - Boss到达基地 → 直接触发「失败」判定（基地HP归零）
  - Boss阶段切换由敌人AI系统处理，波次系统只监听死亡/到达基地事件
```

## Formulas（公式）

**波次补贴（来自资源/货币系统）**
```
波次补贴 = 50 + (wave_number - 1) × 10
Boss波（第10波）无补贴 = 0
```

**波次总时长估算**
```
生成时长 = (敌人总数 - 1) × spawn_interval
示例：波次9共13只 → 生成时长 = 12 × 1.5 = 18秒
战斗时长 ≈ 生成时长 + 最后一只敌人存活时长（取决于塔强度）
```

**累计压力曲线**
```
波次1：5只 × 120hp = 600总HP
波次5：6只 × ~300hp = ~1800总HP
波次9：13只 × ~150hp = ~1950总HP（群体但个体较脆）
Boss波：1只 × 3000hp = 3000总HP（单体但高HP+阶段）
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 玩家在倒计时结束前主动跳过准备时间 | 提供「跳过」按钮，立即结束倒计时进入生成阶段 |
| 敌人生成时spawn_point被塔占据 | spawn_point由关卡设计保证不可放塔；若被推入则临时移至相邻空格 |
| 波次内敌人全部死亡但spawn_list未生成完 | 继续按计划生成剩余敌人（清场判定基于所有敌人，包括未生成的） |
| Boss战途中基地HP归零 | 立即触发失败，不等Boss死亡 |
| 第10波后还有存活敌人（非Boss） | 第10波为纯Boss波，无其他敌人 |
| 菊石群5只同时到达基地 | 各自独立造成伤害，基地HP扣减5次 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| prepare_time | 15秒（第1波30秒） | 10~30秒 | 越长玩家建造时间越充裕；越短节奏越紧张 |
| spawn_interval | 1.5秒 | 0.8~3.0秒 | 越短敌人集中涌来压力越大；越长分散易各个击破 |
| 波次敌人总数增长 | 5→13只（波1→波9） | — | 调整各波数量影响压力曲线斜率 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 每波开始前倒计时15秒（第1波30秒） | 观察游戏开始和波次间隔的倒计时时长 |
| AC-2 | 波次开始时正确发放补贴 | 第3波开始确认获得70金币补贴 |
| AC-3 | 敌人按spawn_interval间隔依次生成 | 计时相邻敌人生成间隔，确认约1.5秒 |
| AC-4 | 所有敌人离场后波次结束 | 击杀最后一只敌人，确认波次结算界面出现 |
| AC-5 | Boss波不发放补贴 | 第10波开始确认金币不增加 |
| AC-6 | Boss死亡触发胜利 | 击杀霸王龙王，确认胜利界面出现 |
| AC-7 | 第1波只生成迅猛龙×5 | 开始游戏，确认第1波只有5只迅猛龙 |
| AC-8 | 菊石群2组共10只正确生成 | 第9波确认出现10只菊石（2组×5） |
