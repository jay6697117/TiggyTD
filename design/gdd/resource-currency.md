## Overview（概述）

资源/货币系统管理 TiggyTD 中所有经济行为。游戏内唯一货币是**金币（Gold）**，用于放置动物塔、商店购买道具、升级技能树。金币来源包括：击败敌人获得奖励、波次开始前的固定补贴、商店出售道具。系统负责金币的收入、支出、余额维护，并对每笔交易进行合法性校验（余额不足时拒绝支出）。

---

## Player Fantasy（玩家幻想）

金币是「决策压力」的载体。玩家永远不够钱——这不是惩罚，而是让每一次放置都有意义。看着金币从10涨到120时那种「终于够了」的满足感，以及「我现在到底是该升级猎豹还是再买一只鹰」的纠结感，是塔防策略深度的核心体验。金币流动要有节奏：波次间的补贴给玩家喘息空间，击败精英敌人的大额奖励给玩家爽感峰值。

## Detailed Design（详细设计）

### 货币种类

| 货币 | 说明 |
|------|------|
| 金币（Gold） | 唯一游戏内货币，局内获得局内消耗，局结束清零 |

> 跨局货币（如「星尘」用于技能树永久升级）属于跨局成长系统，不在本文档范围内。

### 金币来源

| 来源 | 金额 | 触发时机 |
|------|------|----------|
| 击败普通敌人 | EnemyDB.get(id).reward | 敌人HP归零时 |
| 击败精英敌人 | EnemyDB.get(id).reward | 敌人HP归零时 |
| 击败Boss | EnemyDB.get(id).reward | Boss HP归零时 |
| 波次开始补贴 | 基础补贴 + 波次递增 | 每波倒计时结束、新波次开始时 |
| 出售道具 | ItemDB.get(id).sell_price | 玩家在商店出售道具时 |

### 金币消耗

| 消耗 | 金额 | 触发时机 |
|------|------|----------|
| 放置动物塔 | AnimalDB.get(id).cost | 玩家拖放动物到地图时 |
| 商店购买道具 | ItemDB.get(id).buy_price | 玩家点击购买时 |
| 技能树升级 | 各节点定义的cost | 玩家点击技能树节点升级时 |

### 波次补贴公式

```
波次补贴 = base_stipend + (wave_number - 1) × stipend_per_wave
base_stipend = 50（第1波固定补贴）
stipend_per_wave = 10（每波递增）
示例：第1波=50，第2波=60，第3波=70 ... 第10波=140
```

### 余额校验规则

```
支出请求 → 检查 current_gold >= cost
  若 current_gold >= cost：
    current_gold -= cost
    返回 SUCCESS
  若 current_gold < cost：
    返回 INSUFFICIENT_GOLD（不扣款，UI显示红色提示）
```

### 状态

| 字段 | 类型 | 说明 |
|------|------|------|
| current_gold | int | 当前金币余额，最小值0 |
| total_earned | int | 本局累计收入（调试/统计用） |
| total_spent | int | 本局累计支出（调试/统计用） |

## Formulas（公式）

**波次补贴**
```
波次补贴 = 50 + (wave_number - 1) × 10
变量：wave_number = 当前波次编号，范围 1~10（MVP）
示例：
  第1波：50 + 0 = 50金币
  第5波：50 + 40 = 90金币
  第10波：50 + 90 = 140金币
10波总补贴 = Σ(50 + (n-1)×10), n=1~10 = 950金币
```

**放置成本参考**
```
最低成本：变色龙 70金币
最高成本：大象 180金币
平均成本：约 120金币
第1波补贴50金币 → 第1波只能放置低价动物（变色龙/水獭/猎豹）
第3波累计补贴(50+60+70=180)金币 + 敌人奖励 → 可以开始放置高价动物
```

**敌人金币奖励参考**
```
普通敌人：8~20金币/只
精英敌人：18~30金币/只
Boss：200~500金币/只
10波敌人击败总奖励估算（MVP）：约 800~1200金币（取决于关卡设计）
```

**全局经济收支估算（MVP 10波）**
```
总收入 ≈ 波次补贴950 + 敌人奖励1000 = ~1950金币
可放置动物数量估算：1950 ÷ 120（平均成本） ≈ 16只
（含重复放置/替换的情况，实际放置决策数约10~14次）
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 金币不足时尝试放置动物 | 返回INSUFFICIENT_GOLD，UI按钮变红/shake动画，不扣款不放置 |
| 多只敌人同帧死亡 | 每只独立触发奖励，金币累加；不合并为单次结算 |
| Boss分阶段——击败第一阶段是否给奖励 | Boss只在最终HP归零时给reward，阶段切换不给奖励 |
| 波次补贴发放时玩家正在拖动动物 | 补贴立即加入余额；拖动动作完成后检查新余额是否足够 |
| 出售道具后余额超出上限 | 金币无上限，不设最大值（避免「囤钱」无意义） |
| 局结束时金币余额 | 清零，不结转到下局（跨局成长由跨局成长系统另行处理） |
| 技能树升级被取消（退款） | 当前设计不支持退款；升级不可逆（技能树系统定义） |
| current_gold出现负数 | 不应发生（校验保护）；若发生则强制归零并记录错误日志 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| base_stipend（第1波补贴） | 50 | 30~80 | 越高开局越宽裕；越低开局越紧张，考验前期选择 |
| stipend_per_wave（每波递增） | 10 | 5~20 | 越高后期越富裕，缓解后期压力；越低全程紧张 |
| 普通敌人reward范围 | 8~20 | 5~30 | 影响「击败感」和经济节奏，建议与hp成正比 |
| 精英敌人reward范围 | 18~30 | 15~50 | 精英应明显高于普通，强化「打精英值得」的感知 |
| Boss reward | 200~500 | 100~800 | Boss奖励是经济爆发点，太低Boss无存在感，太高破坏经济平衡 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 击败敌人后金币立即增加对应reward值 | 击败reward=15的三角龙，确认current_gold增加15 |
| AC-2 | 每波开始时发放正确补贴 | 第3波开始时确认获得70金币补贴（50+(3-1)×10） |
| AC-3 | 金币不足时放置动物被拒绝 | current_gold=50时尝试放置cost=70的变色龙，确认放置失败且金币不变 |
| AC-4 | 放置成功后金币正确扣除 | current_gold=100时放置cost=70的变色龙，确认current_gold变为30 |
| AC-5 | 出售道具后金币正确增加 | 出售sell_price=40的道具，确认current_gold增加40 |
| AC-6 | 多只敌人同帧死亡金币全部结算 | 同帧击败3只reward=10的迅猛龙，确认current_gold增加30 |
| AC-7 | 局结束后金币清零 | 一局结束后读取current_gold，确认值为0 |
| AC-8 | current_gold不会出现负数 | 任何情况下读取current_gold，确认值≥0 |
