# 伤害计算系统

> **状态**: 设计中
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 支柱二（Build 进化自由）、支柱四（像素特效美学）

---

## Overview（概述）

伤害计算系统是 TiggyTD 所有伤害事件的统一计算入口。每当动物塔、英雄技能或陷阱对史前生物造成伤害时，系统接收攻击方的基础伤害、属性修正和状态标签，按照固定公式计算最终伤害值，扣除目标 HP，并向视觉反馈系统输出伤害数字和命中类型标签（普通/暴击/无视护甲/反伤）。该系统是 Build 深度的数值基础——技能树路线和道具合成的价值，最终都通过本系统的输出差异体现给玩家。

---

## Player Fantasy（玩家幻想）

**玩家幻想**：「我的 Build 是独一无二的武器。」

玩家应该能清晰感受到：同样是犀牛，走「刺甲流」之后反弹伤害数字飞出来的那一刻，和「盾墙流」的感觉完全不同。暴击时弹出的金色大数字、无视护甲时的穿透特效、反伤触发时的红色反弹数字——每一种命中类型都有视觉语言，让玩家**不需要看数值面板**也能感受到 Build 在生效。

核心感受优先级：
1. **Build 有效感**：技能树投资和道具合成要在伤害数字上有明显体现
2. **时机爽感**：暴击、反伤、协同触发时的视觉反馈要比普通攻击明显放大
3. **策略验证感**：用针对性 Build 打 Boss 弱点时，「+100% 弱点伤害」应该有专属的视觉表现

## Detailed Design（详细设计）

### Core Rules（核心规则）

所有伤害事件通过统一入口计算，输出最终伤害值和命中类型标签。

**主公式**
```
最终伤害 = 基础伤害 × 护甲修正 × 暴击修正 × 增益修正 × 弱点修正
```

**基础伤害**
```
基础伤害 = 动物塔基础攻击力 × (1 + 技能树攻击加成) + 道具附加伤害
```
- `动物塔基础攻击力`：由动物数据库定义
- `技能树攻击加成`：范围 0.0 ~ 3.0
- `道具附加伤害`：固定值，由道具数据库定义

**护甲修正**
```
有效护甲率 = 敌人护甲率 × (1 - 穿透率)
护甲修正 = max(0.25, 1 - 有效护甲率)
```
- `敌人护甲率`：范围 0.0 ~ 0.75，由敌人数据库定义
- `穿透率`：范围 0.0 ~ 1.0，通过技能树/道具获得
- 蜜獾「无视护甲」= 穿透率强制为 1.0
- 护甲修正下限 0.25（最高减免75%）

**暴击修正**
```
触发条件：random() < 暴击率
暴击修正 = 1 + 暴击倍率（触发时）
暴击修正 = 1.0（未触发时）
```
- `暴击率`：基础5%，上限75%
- `暴击倍率`：基础0.5，技能树可提升至3.0（即最高4.0x伤害）
- 百兽怒吼大招：暴击率临时+30%（叠加）

**增益修正**
```
增益修正 = (1 + 增益1) × (1 + 增益2) × ... 上限 10.0
```
- 所有增益以乘法叠加
- 例：老虎光环+20% × 孔雀魅惑+20% = 1.2 × 1.2 = 1.44x

**弱点修正**
```
弱点修正 = 2.0（Boss 被「纪元审判」标记后5秒内）
弱点修正 = 1.0（默认）
```

**特殊伤害类型**

反弹伤害（犀牛刺甲流）：
```
反弹伤害 = 受到伤害值 × 反弹率
```
- 不触发暴击，不受护甲修正，直接扣除敌人HP
- `反弹率`：基础0.3，技能树可提升至1.0

陷阱伤害（变色龙）：
```
陷阱伤害 = 基础陷阱伤害 × 增益修正
```
- 不触发暴击（陷阱为固定伤害）

### States and Transitions（状态与转换）

| 状态 | 说明 |
|------|------|
| IDLE | 无伤害事件 |
| CALCULATING | 收到伤害请求，执行公式 |
| APPLYING | 扣除目标HP，输出命中类型标签 |
| NOTIFYING | 通知视觉反馈系统显示伤害数字 |

每次攻击命中触发一次完整的 CALCULATING → APPLYING → NOTIFYING 流程，同帧多次命中（如狼群）各自独立计算。

### Interactions with Other Systems（与其他系统的交互）

| 系统 | 交互方式 |
|------|----------|
| 动物塔攻击系统 | 提供基础攻击力、攻击加成、穿透率、暴击率/倍率 |
| 状态效果系统 | 提供增益/减益修正值，弱点标记状态 |
| 道具装备系统 | 提供道具附加伤害和穿透率加成 |
| 敌人数据库 | 提供目标护甲率和HP |
| 特效/视觉反馈系统 | 接收命中类型标签，显示对应伤害数字样式 |
| 碰撞检测系统 | 提供命中事件触发信号（provisional：命中=攻击范围内检测到目标） |

## Formulas（公式）

### 主公式
```
最终伤害 = 基础伤害 × 护甲修正 × 暴击修正 × 增益修正 × 弱点修正
```

### 变量定义与范围

| 变量 | 类型 | 范围 | 来源 |
|------|------|------|------|
| 动物塔基础攻击力 | float | 5 ~ 200 | 动物数据库 |
| 技能树攻击加成 | float | 0.0 ~ 3.0 | 动物技能树系统 |
| 道具附加伤害 | float | 0 ~ 100 | 道具数据库 |
| 敌人护甲率 | float | 0.0 ~ 0.75 | 敌人数据库 |
| 穿透率 | float | 0.0 ~ 1.0 | 技能树/道具 |
| 暴击率 | float | 0.05 ~ 0.75 | 技能树/道具/buff |
| 暴击倍率 | float | 0.5 ~ 3.0 | 技能树 |
| 增益修正 | float | 1.0 ~ 10.0 | 状态效果系统 |
| 弱点修正 | float | 1.0 or 2.0 | 状态效果系统 |
| 反弹率 | float | 0.3 ~ 1.0 | 技能树 |

### 示例计算

**示例1：普通犀牛攻击史前野猪**
```
基础攻击力 = 30，技能树加成 = 0.5，道具附加 = 10
基础伤害 = 30 × (1 + 0.5) + 10 = 55
敌人护甲率 = 0.3，穿透率 = 0
护甲修正 = 1 - 0.3 = 0.7
暴击未触发 → 暴击修正 = 1.0
无增益 → 增益修正 = 1.0
无弱点 → 弱点修正 = 1.0
最终伤害 = 55 × 0.7 × 1.0 × 1.0 × 1.0 = 38.5 → 取整 38
```

**示例2：蜜獾无视护甲暴击攻击**
```
基础攻击力 = 40，技能树加成 = 1.0，道具附加 = 20
基础伤害 = 40 × 2.0 + 20 = 100
穿透率 = 1.0（无视护甲）
有效护甲率 = 0.5 × (1 - 1.0) = 0
护甲修正 = max(0.25, 1 - 0) = 1.0
暴击触发，暴击倍率 = 2.0 → 暴击修正 = 1 + 2.0 = 3.0
老虎光环增益+20% → 增益修正 = 1.2
最终伤害 = 100 × 1.0 × 3.0 × 1.2 × 1.0 = 360
```

**示例3：纪元审判标记后打 Boss**
```
基础伤害 = 80，护甲修正 = 0.5，暴击修正 = 1.0，增益修正 = 1.44
弱点修正 = 2.0
最终伤害 = 80 × 0.5 × 1.0 × 1.44 × 2.0 = 115.2 → 取整 115
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 最终伤害计算结果 < 1 | 强制输出1（任何攻击至少造成1点伤害） |
| 增益修正超过上限10.0 | 截断至10.0，不报错 |
| 暴击率超过75% | 截断至0.75，多余的加成无效 |
| 穿透率 > 1.0 | 截断至1.0（不能出现负护甲） |
| 反弹伤害对象已死亡 | 丢弃反弹事件，不执行 |
| 同一帧多个增益同时到期 | 全部纳入本帧计算，下帧移除 |
| 弱点修正与暴击同时触发 | 两者均生效，正常乘法叠加 |
| 陷阱被多只敌人同时踩中 | 每只敌人各自独立触发一次陷阱伤害计算 |
| 老虎大招百兽怒吼期间暴击率超过75% | 截断至0.75，超出部分无效 |
| 伤害结果为浮点数 | 所有最终伤害向下取整（floor），保持整数显示 |

- **碰撞检测系统**：命中判定结果以事件形式传入本系统，事件包含：攻击方ID、目标ID、攻击类型标签。碰撞系统设计时必须对齐此接口。
- **状态效果系统**：增益/减益以修正值列表形式传入，本系统负责乘法叠加。状态效果系统不直接修改HP，只提供修正参数。

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 暴击率上限 | 75% | 60% ~ 85% | 超过85%暴击变成「必然」，失去紧张感；低于60%暴击Build体验不足 |
| 暴击倍率上限 | 3.0（4.0x） | 2.0 ~ 4.0 | 超过4.0后期数值膨胀严重；低于2.0暴击Build不够爽 |
| 护甲率上限 | 75% | 60% ~ 80% | 超过80%穿透Build几乎是必选；低于60%护甲感不明显 |
| 护甲修正下限 | 0.25 | 0.15 ~ 0.35 | 影响穿透率100%时的最大伤害放大倍数 |
| 增益修正上限 | 10.0 | 8.0 ~ 15.0 | 防止协同叠加后数值爆炸，低于8.0会限制极端Build的爽感 |
| 反弹率上限 | 1.0 | 0.8 ~ 1.2 | 超过1.0反伤超过受到伤害，需配合敌人伤害上限一起调 |
| 弱点修正倍数 | 2.0 | 1.5 ~ 3.0 | 影响「纪元审判」的Boss收割感，低于1.5弱点机制不明显 |
| 基础暴击率 | 5% | 3% ~ 10% | 影响早期无Build时的随机爽感频率 |
| 基础暴击倍率 | 0.5（1.5x） | 0.3 ~ 1.0 | 影响无Build投资时的暴击体验下限 |

## Acceptance Criteria（验收标准）

以下每条标准必须可由 QA 独立验证通过：

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 任何攻击造成的最终伤害 ≥ 1 | 用护甲率0.75、穿透率0、最低攻击力攻击，确认输出 ≥ 1 |
| AC-2 | 蜜獾攻击无视护甲，护甲修正 = 1.0 | 对护甲率0.75的敌人，蜜獾与同属性普通单位伤害对比，蜜獾应高出约4倍 |
| AC-3 | 暴击率超过75%时截断，不超过75% | 将暴击率堆到100%，实测触发频率应 ≈ 75% |
| AC-4 | 百兽怒吼期间全场暴击率+30%叠加生效 | 记录大招前后暴击触发频率，差值应 ≈ 30% |
| AC-5 | 增益修正超过10.0时截断 | 叠加11个+100%增益，最终增益修正应 = 10.0 |
| AC-6 | 弱点修正触发时伤害翻倍 | 对Boss标记弱点前后各打10次，平均伤害差应 ≈ 2.0x |
| AC-7 | 反弹伤害不受护甲修正影响 | 犀牛受到100伤害，反弹率0.3，敌人扣除HP应 = 30（不经护甲） |
| AC-8 | 伤害数字命中类型标签正确 | 触发暴击时显示金色数字；触发弱点时显示紫色数字；反伤显示红色数字 |
| AC-9 | 同帧多次命中各自独立计算 | 狼群3只同时命中同一敌人，敌人HP扣除值 = 3次独立计算结果之和 |
| AC-10 | 浮点伤害向下取整 | 计算结果38.7应显示并扣除38，不得四舍五入 |
