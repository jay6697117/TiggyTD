# 动物塔攻击系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 战斗核心（动物塔的攻击行为与目标选择）

---

## Overview（概述）

动物塔攻击系统驱动所有已放置动物塔的攻击行为：目标搜索、目标选择策略、攻击间隔计时、伤害输出。系统读取动物数据库的攻击属性，调用伤害计算系统输出最终伤害，调用寻路系统获取路径进度（用于First策略）。每个塔实例维护独立的攻击计时器和当前目标引用。

---

## Player Fantasy（玩家幻想）

动物塔攻击是「Build生效」的直接体现。猎豹快速连击的爽快感、鹰从高空俯冲的冲击感、狼群三路夹击的协同感——每种动物的攻击应该有独特的视觉语言和节奏感。玩家看到猎豹对迅猛龙快速输出时感到满足，看到穿山甲的高护甲穿透对腕龙造成巨额伤害时感到「Build奏效了」的爽感。

## Detailed Design（详细设计）

### 攻击循环

```
每帧更新（每个塔实例）：
1. 目标搜索：在range范围内查找所有敌人
2. 目标选择：按当前策略选出优先目标
3. 攻击计时：attack_timer -= delta
4. 若attack_timer ≤ 0 且有有效目标：
   a. 调用 DamageSystem.calculate(atk, target.armor)
   b. 对目标造成伤害
   c. attack_timer = 1.0 / atk_speed（重置计时器）
5. 若无有效目标：保持IDLE，计时器不重置
```

### 目标选择策略

| 策略 | 说明 | 实现 |
|------|------|------|
| First（默认） | 优先攻击路径进度最靠前的敌人 | 查询寻路系统的路径进度，取最大值 |
| Closest | 优先攻击物理距离最近的敌人 | 取distance最小值 |
| Weakest | 优先攻击当前HP最低的敌人 | 取hp最小值 |
| Strongest | 优先攻击当前HP最高的敌人 | 取hp最大值 |

> 默认策略为First。玩家可在塔的UI面板中切换策略（游戏UI系统负责交互）。

### 攻击范围检测

```
范围检测：以塔中心为圆心，range格为半径的圆形区域
检测层级：ENEMY碰撞层（来自碰撞检测系统）
翼龙特殊：只有range≥3的塔才能命中翼龙（fly被动，敌人AI系统标记）
```

### 狼群特殊处理

```
狼群由3个独立攻击单位组成，各自维护独立的：
  - 攻击计时器
  - 当前目标
  - 碰撞体
狼群作为一个「塔单位」放置，但内部有3个子单位各自攻击
死亡判定：3只全部死亡 → 塔单位移除
```

### 状态机

| 状态 | 说明 |
|------|------|
| IDLE | 范围内无目标，等待 |
| ATTACKING | 有目标，按atk_speed循环攻击 |
| STUNNED | 被状态效果眩晕，停止攻击（状态效果系统控制） |

## Formulas（公式）

**攻击间隔**
```
attack_interval = 1.0 / atk_speed（秒）
示例：猎豹atk_speed=2.5 → 每0.4秒攻击一次
示例：大象atk_speed=0.5 → 每2.0秒攻击一次
```

**每秒伤害（DPS，不含护甲）**
```
DPS = base_atk × atk_speed
猎豹：25 × 2.5 = 62.5 DPS
鹰：50 × 1.0 = 50 DPS
狼群：15 × 1.5 × 3 = 67.5 DPS（三只合计）
穿山甲：40 × 0.7 = 28 DPS（但穿透护甲，对高甲目标实际更高）
```

**实际伤害（含护甲，调用伤害计算系统）**
```
实际伤害 = DamageSystem.calculate(base_atk, target.armor)
见 damage-calculation.md
```

**First策略路径进度**
```
路径进度 = 已经过路径点数量 + (1 / 距下一路径点距离)
取范围内所有敌人中路径进度最大的作为目标
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 当前目标死亡时攻击动画未结束 | 动画继续播放，但伤害不结算（目标已不存在）；下帧重新选目标 |
| 范围内只有翼龙但塔range<3 | 无有效目标，进入IDLE |
| 攻击目标在攻击瞬间离开range | 伤害仍然结算（攻击已触发）；下帧重新选目标 |
| 狼群一只被击杀 | 该只从战场移除，另两只继续攻击；塔单位未消除 |
| 攻击目标被ice_armor减伤 | 伤害计算系统输出减半后的值，攻击系统直接使用 |
| 多个塔同帧击杀同一敌人 | 第一个伤害触发reward和死亡，后续伤害对已死亡敌人无效 |
| STUNNED状态下新目标进入range | 眩晕结束后自动开始攻击，STUNNED期间不重置计时器 |
| 塔被三角龙推开时正在攻击 | 中断攻击，位移完成后以新位置重新搜索目标 |

## Dependencies（依赖关系）

### 上游依赖

| 系统 | 使用内容 |
|------|----------|
| 动物数据库 | base_atk、atk_speed、range等攻击属性 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 猎豹atk_speed | 2.5 | 1.5~3.5 | 越高猎豹DPS越强，是主力DPS调节旋钮 |
| 鹰range | 7.0格 | 5.0~9.0 | 越大鹰的覆盖越广，是远程单位调节旋钮 |
| 狼群子单位数量 | 3只 | 2~4只 | 越多狼群DPS越高，但AI和碰撞体开销增加 |
| 穿山甲护甲穿透率 | 见伤害计算系统 | — | 在damage-calculation.md中调节 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 塔按atk_speed正确间隔攻击 | 猎豹(atk_speed=2.5)在10秒内攻击约25次 |
| AC-2 | First策略优先攻击路径进度最靠前的敌人 | 两只敌人在路径不同位置，确认塔攻击进度更靠前的 |
| AC-3 | range<3的塔不攻击翼龙 | 在range=2的塔旁放翼龙，确认塔进入IDLE不攻击 |
| AC-4 | range≥3的塔正常攻击翼龙 | 鹰(range=7)在翼龙range内，确认正常攻击 |
| AC-5 | 目标死亡后立即切换新目标 | 当前目标被击杀后，确认塔在下一帧攻击新目标 |
| AC-6 | 狼群3只各自独立攻击不同目标 | 范围内有3只敌人时，确认狼群3只分别攻击不同敌人 |
| AC-7 | 一只狼死亡后另两只继续攻击 | 击杀狼群中一只，确认另两只继续正常攻击 |
| AC-8 | 实际伤害通过伤害计算系统计算 | 攻击armor=30%的三角龙，确认伤害值符合damage-calculation.md公式 |
