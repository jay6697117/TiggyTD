# 游戏支柱文档

**项目代号**：TiggyTD
**版本**：v1.0

---

## 支柱一：图腾指挥感

**核心体验**：老虎不只是一个角色，是玩家意志的化身。每次大招释放都是「我决定了」的仪式感时刻。

**设计含义**：
- 老虎的存在必须在战场上有持续的可见价值，而不只是「偶尔按一下技能」
- 全局 buff 光环让玩家感受到「我在场，队伍就更强」
- 大招的冷却时间设计要制造「忍耐 → 释放」的张力节奏
- 英雄移动本身应该是有意义的决策，而不是随意走动

**违反此支柱的设计**：老虎只是装饰、大招可以随意乱按没有时机感、英雄在战场上可有可无。

## 支柱二：Build 进化自由

**核心体验**：同一只动物在不同技能树路线下可以变成完全不同的战士。每局的 Build 探索是核心乐趣，「再来一局」的冲动来自「我想试试另一条路线」。

**设计含义**：
- 技能树路线分叉要足够明显，让玩家感受到「选了这条就回不了头」的决策重量
- 道具合成系统要有「差一件就能合成」的钩子感
- 不同 Build 应该产生截然不同的视觉效果（反映在特效上）
- 平衡性设计目标：没有「最强 Build」，只有「最适合这关的 Build」

**违反此支柱的设计**：某一路线明显最优、技能树节点差异微小、Build 之间的差异只有数字而没有玩法变化。

---

## 支柱三：纪元对抗叙事

**核心体验**：现代 vs 史前不只是视觉包装，而是策略层面的对称。史前生物有行为逻辑，玩家需要读懂并反制，产生「我看穿你了」的智识满足感。

**设计含义**：
- 每种史前生物的特殊能力应该有明确的「反制方法」
- Boss 战要有「这局我特意为你准备了」的针对性 Build 体验
- 入侵主题（每关的史前阵营）要影响玩家的 Build 选择方向
- 世界观叙事通过关卡设计传达，不需要大量文字

**违反此支柱的设计**：史前生物只是血量更高的普通怪、Boss 没有弱点机制、每关敌人差异只有数值。

---

## 支柱四：像素特效美学

**核心体验**：高清唯美像素 + 日系卡通风格，特效是游戏的视觉语言。每个技能释放都要有「值回票价」的画面感，让玩家愿意截图分享。

**设计含义**：
- 特效质量是第一优先级的美术投入方向
- 每只动物的技能特效要有辨识度，体现动物本身的个性
- 老虎大招特效要明显优于普通技能，体现「王者」地位
- 像素风格不等于低质量，要追求「精致的像素」而非「粗糙的像素」
- 日系卡通色彩搭配：饱和度高、对比鲜明、色调温暖

**违反此支柱的设计**：所有技能特效雷同、像素分辨率不一致、色彩风格混乱。

## 支柱优先级排序

当设计决策发生冲突时，按以下优先级取舍：

1. **Build 进化自由**（核心玩法深度，决定游戏的重玩价值）
2. **图腾指挥感**（差异化体验，区别于普通塔防的核心）
3. **像素特效美学**（第一印象和传播钩子）
4. **纪元对抗叙事**（丰富层次，但不是基础体验的必要条件）

## 一句话检验标准

每次做设计决策时，问自己：
> **「这个设计让玩家感受到：指挥万兽抵御史前入侵、每局 Build 都是我的独创杰作了吗？」**

如果答案是否，重新考虑。
