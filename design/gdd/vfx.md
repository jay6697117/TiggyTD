# 特效/视觉反馈系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27

---

## Overview（概述）

特效/视觉反馈系统定义游戏中所有像素风格粒子特效、动画反馈和屏幕效果的规格与触发时机。系统本身不含独立逻辑——它是其他所有系统的「感知层」，当战斗事件、状态变化、UI交互发生时，对应的VFX被调用。核心目标是让每个玩家操作和游戏事件都有清晰、即时、像素美学一致的视觉反馈。

---

## Player Fantasy（玩家幻想）

像素特效美学是游戏四大支柱之一。猎豹暴击时爆出金色像素碎片、大象踩踏时地面出现裂纹波纹、纪元审判落地时全屏像素震动——这些特效让战斗「有重量」、让玩家操作「有回应」。好的VFX系统让玩家不看数字也能感知战场状态：蓝色粒子=减速中，绿色粒子=中毒，紫色能量=Boss护盾激活。

### VFX 调用接口

```
// 其他系统调用VFX的统一接口
VFXSystem.play(effect_id, position, [duration_override], [color_override])

// 示例
VFXSystem.play("hit_normal", enemy.position)
VFXSystem.play("status_slow", enemy.position)  // 持续效果，跟随目标
VFXSystem.play("skill_era_judgement_hit", target.position)
```

### 像素风格规范

```
粒子最小单位：1×1像素 或 2×2像素（保持像素感）
不使用模糊/渐变滤镜（保持清晰像素边缘）
颜色数量：每个特效最多使用3种颜色（保持简洁）
帧率：特效动画以8fps播放（像素动画质感）
```

## Formulas（公式）

**屏幕震动衰减**
```
shake_offset(t) = intensity × (1 - t/duration) × sin(t × 40)
其中 t=当前时间（秒），intensity=震动强度（像素），duration=震动持续时间
普通扣血：intensity=5, duration=0.3
Boss登场：intensity=10, duration=2.0
纪元审判命中：intensity=8, duration=0.8
```

**伤害数字飘动**
```
初始位置：命中点
每帧Y偏移：-2像素（向上）
存活帧数：0.8秒 × 60fps = 48帧
透明度：前24帧=1.0，后24帧线性衰减至0
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 同帧多个命中特效重叠 | 每个特效独立播放，允许视觉重叠（像素风下反而增加爽快感） |
| 持续状态特效目标死亡 | 目标死亡时销毁绑定的持续特效实例 |
| 大量敌人同时死亡（波次清场）特效过多 | 同帧超过10个死亡特效时，随机跳过50%的特效播放（防止卡帧） |
| 屏幕震动叠加（多次触发） | 取最大intensity值，重置计时器，不累加 |
| 低性能设备 | 提供「简化特效」选项：关闭持续粒子，只保留命中闪光 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 普通命中粒子数量 | 6~8个 | 4~12个 | 越多打击感越强，性能开销越大 |
| 伤害数字存活时间 | 0.8秒 | 0.5~1.5秒 | 越长屏幕越拥挤（密集战斗时） |
| 屏幕震动强度（扣血） | 5px | 0~15px | 越强冲击感越强，过强令人不适 |
| 特效帧率 | 8fps | 6~12fps | 越高越流畅，越低像素感越强 |
| 同帧最大死亡特效数 | 10个 | 5~20个 | 越多清场视觉越爽，越少性能越稳定 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 普通攻击命中出现白色像素碎片特效 | 猎豹攻击迅猛龙，确认命中点出现6~8个白色像素粒子扩散 |
| AC-2 | 暴击命中出现金色大型特效，伤害数字金色 | 触发猎豹暴击，确认金色闪光且数字为金色 |
| AC-3 | 减速状态目标持续显示蓝色雪花粒子 | 树懒命中敌人后，确认敌人周围持续蓝色粒子直到减速结束 |
| AC-4 | 百兽怒吼施放时金色冲击波从英雄扩散到屏幕边缘 | 使用百兽怒吼，确认冲击波动画在0.6秒内完成 |
| AC-5 | 基地扣血时屏幕边缘红色闪烁+震动 | 让敌人到达终点，确认屏幕边缘红闪和5px震动 |
| AC-6 | Boss登场触发10px屏幕震动持续2秒 | 第10波开始，确认屏幕震动强度和持续时间 |
| AC-7 | 同帧超过10个死亡特效时随机跳过50% | 触发清场（>10只敌人同时死亡），确认帧率不明显下降 |
| AC-8 | 所有粒子保持像素风格（无模糊/渐变） | 截图放大检查，确认粒子边缘清晰，无抗锯齿模糊 |
