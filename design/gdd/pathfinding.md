# 寻路系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 技术基础（支撑敌人移动和英雄移动）

---

## Overview（概述）

寻路系统为 TiggyTD 中所有需要导航的单位提供路径计算服务。敌人沿预设路径点移动（固定路线），英雄老虎在地图上自由移动（动态寻路）。系统基于 Godot 内置的 NavigationServer2D，格子地图使用 A* 算法计算最短路径。当地图状态改变（塔被推开、障碍物移除）时，系统自动触发受影响区域的路径重算。

---

## Player Fantasy（玩家幻想）

寻路系统是「世界感」的基础设施。敌人沿路径涌来的压迫感、英雄在战场上灵活穿梭的操控感，都依赖寻路系统的精确运行。玩家不会直接感知到它，但三角龙冲撞推开塔后敌人立刻绕路的「智能感」，以及老虎点击目标位置后流畅移动的「响应感」，都是寻路质量的直接体现。

## Detailed Design（详细设计）

### Core Rules（核心规则）

#### 两种寻路模式

**模式1：敌人固定路线（Waypoint Path）**

敌人沿关卡设计师预设的路径点序列移动，不使用动态A*计算。

```
路径点序列：[spawn_point → waypoint_1 → waypoint_2 → ... → base]
敌人移动：逐点移动，到达当前路径点后自动前往下一个
速度：EnemyDB.get(id).speed（格/秒）
减速效果：speed × (1 - 减速率)
```

路径点之间使用**直线插值**移动，不绕障碍（路径点本身已由关卡设计师规避障碍）。

**路径重算触发条件**（仅影响动态障碍）：
- 三角龙冲撞推开动物塔（塔占据新格子，原格子释放）
- 动物塔被摧毁（格子释放）
- 新放置动物塔（格子被占据，但塔不阻挡敌人路径——敌人路径预设绕过塔放置区）

> **设计约定**：关卡地图设计时，敌人路径与塔放置区域分离。敌人走固定通道，塔放置在通道两侧。三角龙冲撞是唯一可能阻塞路径的情况，触发局部A*重算。

**模式2：英雄自由寻路（Dynamic A*）**

英雄老虎根据玩家点击目标位置，使用A*算法计算最短路径。

```
输入：当前位置、目标位置
算法：A*（启发函数：曼哈顿距离）
网格：与地图格子对齐，每格1.0单位
障碍物：地形边界、水域（英雄不可进入）
动物塔：英雄可穿越（不同碰撞层级，来自碰撞检测系统）
输出：Vector2路径点数组
重算触发：目标位置变更、障碍物状态改变
```

#### 寻路网格定义

| 格子类型 | 英雄可通行 | 敌人可通行 | 备注 |
|---------|-----------|-----------|------|
| 普通地面 | ✅ | ✅（固定路径上） | 标准移动 |
| 水域 | ❌ | ✅（沧龙速度×1.5） | 英雄不可进入 |
| 地形边界 | ❌ | ❌ | 地图边缘 |
| 塔占据格 | ✅ | ❌（触发绕路） | 英雄穿越塔 |
| 陷阱格 | ✅ | ✅（触发陷阱） | 不阻挡移动 |

#### 目标选择策略（动物塔攻击目标）

塔攻击时需要选择范围内的目标，寻路系统提供「距离终点最近」的排序支持：

| 策略 | 说明 | 默认 |
|------|------|------|
| 最近终点（First） | 优先攻击路径进度最靠前的敌人 | ✅ 默认 |
| 最近距离（Closest） | 优先攻击物理距离最近的敌人 | 可选 |
| 最低HP（Weakest） | 优先攻击HP最低的敌人 | 可选 |
| 最高HP（Strongest） | 优先攻击HP最高的敌人 | 可选 |

### States and Transitions（状态与转换）

| 状态 | 说明 |
|------|------|
| FOLLOWING_WAYPOINTS | 敌人正在沿路径点序列移动（正常状态） |
| RECALCULATING | 路径重算中（三角龙冲撞后，通常<1帧） |
| BLOCKED | 路径被完全阻断（理论上不应发生，触发警告） |
| HERO_NAVIGATING | 英雄正在执行A*路径导航 |
| HERO_IDLE | 英雄到达目标位置，停止导航 |

### Interactions with Other Systems（与其他系统的交互）

| 系统 | 交互方式 |
|------|----------|
| 碰撞检测系统 | 提供障碍物格子信息（地形边界、水域）用于A*网格构建 |
| 敌人AI系统 | 消费路径点序列，驱动敌人移动；接收减速状态修改speed |
| 英雄移动系统 | 消费A*路径结果，驱动英雄移动 |
| 关卡/地图系统 | 提供路径点序列定义、地图格子类型、水域位置 |
| 动物塔放置系统 | 塔被推开时通知寻路系统触发局部路径重算 |
| 状态效果系统 | 减速效果通过修改敌人speed参数影响移动速度，不修改路径本身 |

## Formulas（公式）

**A* 启发函数（曼哈顿距离）**
```
h(node) = |node.x - goal.x| + |node.y - goal.y|
f(node) = g(node) + h(node)  // g=已走代价, h=预估代价
```

**敌人移动速度（含减速）**
```
实际speed = EnemyDB.get(id).speed × (1 - 减速率)
减速率来源：状态效果系统，范围 0.0~0.8（最多减速80%）
沧龙水域加速：实际speed = base_speed × 1.5（水域格子上）
```

**路径进度（用于目标选择策略）**
```
路径进度 = 已经过路径点数量 + (当前位置到下一路径点距离的倒数)
// 进度越高 = 越靠近终点 = First策略优先攻击
```

**英雄A*路径平滑**
```
// 移除共线路径点，减少不必要的转向
如果 waypoint[i-1]、waypoint[i]、waypoint[i+1] 在同一直线上：
  移除 waypoint[i]
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 三角龙冲撞将塔推入路径中间 | 触发局部A*重算，敌人绕路；若无替代路径，触发BLOCKED警告 |
| 塔被推入已有塔占据的格子 | 碰撞检测系统处理推开动画，寻路等待动画结束后收到通知再重算 |
| 英雄点击水域为目标 | A*不返回路径，英雄停在距目标最近的可通行格子 |
| 英雄点击地形边界外 | 点击被忽略，英雄保持当前位置 |
| 多只敌人同时触发重算 | 所有重算请求合并为一次全局局部重算 |
| 敌人在重算期间到达路径点 | 敌人进入RECALCULATING状态暂停，通常<1帧，视觉不可感知 |
| 路径平滑移除过多路径点 | 仅移除共线点，转向点保留；最少保留起点和终点 |
| 英雄A*路径中途目标改变 | 立即丢弃当前路径，以新目标重新计算 |
| 沧龙水域加速与减速同时生效 | 实际speed = base_speed × 1.5 × (1 - 减速率) |
| 路径点间距超出速度单位 | 每帧移动量 = speed × delta，不会跳过路径点 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| A*网格格子大小 | 1.0单位/格 | 0.5~2.0 | 格子越小路径越精确但计算量越大 |
| 曼哈顿启发权重 | 1.0（标准A*） | 1.0~1.5 | >1.0为加权A*，速度更快但路径非最优 |
| 路径重算最大帧延迟 | 1帧 | 1~3帧 | 延迟越低敌人绕路越及时 |
| First策略路径进度倒数权重 | 1.0 | 0.5~2.0 | 影响下一路径点距离对进度排名的贡献 |
| 沧龙水域速度倍率 | 1.5× | 1.2~2.0 | 低于1.2水域优势不明显；高于2.0沧龙几乎无法被拦截 |
| 减速率上限 | 80%（0.8） | 0.5~0.9 | 上限太低减速无意义；上限太高敌人完全停止 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 敌人沿预设路径点序列移动，不偏离路线 | 生成敌人，观察其经过所有路径点后到达基地 |
| AC-2 | 三角龙推开塔后敌人在1帧内重算路径 | 三角龙冲撞推开路径上的塔，确认敌人立即绕路 |
| AC-3 | 英雄点击目标位置后沿A*路径移动 | 点击任意可通行位置，确认英雄走折线路径而非直线穿越障碍 |
| AC-4 | 英雄无法进入水域 | 点击水域格子，确认英雄停在水域边缘的可通行格子 |
| AC-5 | 英雄可穿越动物塔占据的格子 | 英雄A*路径经过塔所在格子，确认正常通过 |
| AC-6 | 沧龙在水域速度×1.5 | 记录沧龙在陆地和水域的移动速度，确认水域速度为陆地×1.5 |
| AC-7 | 减速效果正确降低敌人速度 | 对speed=10的敌人施加50%减速，确认实际速度变为5 |
| AC-8 | First策略选择路径进度最靠前的敌人 | 两只敌人在路径不同位置，塔应优先攻击进度更靠前的敌人 |
| AC-9 | 英雄A*路径平滑（共线点被移除） | 在开阔地带移动，确认路径为直线（无多余折点） |
| AC-10 | BLOCKED状态触发警告日志 | 人为构造完全封闭路径，确认系统输出BLOCKED警告且不崩溃 |
