# 音效系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27

---

## Overview（概述）

音效系统管理游戏内所有音频的播放，包括音效（SFX）和背景音乐（BGM）。音频资源来源为免费授权素材库或AI生成，采用事件驱动方式：游戏事件触发对应音频播放。系统提供音量控制（主音量、SFX音量、BGM音量），设置持久化存储于存档系统。本系统是 Alpha 阶段低优先级实现项。

---

## Player Fantasy（玩家幻想）

音效是视觉反馈的听觉镜像——猎豹攻击的「嗖」声、大象踩踏的低沉「轰」、Boss登场的震撼鼓点，让玩家不看屏幕也能感知战场节奏。恰到好处的音效把像素特效的打击感再放大一倍。BGM从build阶段的轻松旋律过渡到波次进行中的紧张节拍，用音乐驯化玩家的情绪节奏。

## Detailed Design（详细设计）

### 音效事件列表（SFX）

| 事件 | 音效描述 | 优先级 |
|------|---------|--------|
| 普通攻击命中 | 短促打击声（不同动物有差异化版本） | 高 |
| 暴击命中 | 更重的打击声+金属感 | 高 |
| 敌人死亡 | 短促消散音效 | 中 |
| Boss死亡 | 长爆炸音效（3秒） | 高 |
| 英雄技能施放（3个各不同） | 特色音效（怒吼/自然/能量） | 高 |
| 基地扣血 | 低沉警报音 | 高 |
| 金币获得 | 清脆硬币声 | 中 |
| 道具购买/装备 | 满足感短音效 | 中 |
| 技能CD就绪 | 提示音（避免错过技能窗口） | 中 |
| 协同激活 | 上扬弦乐短句 | 中 |
| 胜利 | 胜利号角+欢快旋律（5秒） | 高 |
| 失败 | 低沉失落音效（2秒） | 高 |
| Boss登场 | 震撼鼓点+低频轰鸣 | 高 |
| 升级（英雄天赋） | 清亮上升音效 | 中 |

### BGM 列表

| 场景 | BGM描述 | 循环 |
|------|---------|------|
| 主菜单 | 轻松像素风旋律 | 是 |
| Build阶段 | 轻快策略感背景乐 | 是 |
| 波次进行中 | 紧张节拍感战斗音乐 | 是 |
| Boss战（第10波） | 高强度Boss战专属BGM | 是 |
| 胜利画面 | 胜利主题曲（配合胜利特效） | 否 |
| 失败画面 | 失落短曲 | 否 |

### 音频播放接口

```
// SFX播放（一次性）
AudioSystem.play_sfx(sfx_id, [volume_scale=1.0], [position])
  // position 用于3D距离衰减（MVP暂不实现，所有音效全局音量）

// BGM切换（带淡入淡出）
AudioSystem.play_bgm(bgm_id, [fade_duration=1.0])
  // 当前BGM淡出 fade_duration 秒，新BGM淡入

// 停止BGM
AudioSystem.stop_bgm([fade_duration=0.5])
```

### 音量控制

```
设置项：
  主音量（master_volume）：0~100，默认80
  SFX音量（sfx_volume）：0~100，默认80
  BGM音量（bgm_volume）：0~100，默认60

实际播放音量 = master_volume × sfx_volume（或bgm_volume） / 10000
存储于存档系统 settings.audio 字段
```

### 音频资源规格

```
SFX格式：WAV 或 OGG，单声道，44100Hz，时长<2秒
BGM格式：OGG，立体声，44100Hz，带循环点标记
总资源大小目标：<50MB（避免包体过大）
来源：freesound.org（CC0授权）或 AI生成（Suno/Udio）
```

## Formulas（公式）

**SFX并发上限**
```
同帧最多播放 8 个SFX（防止音频爆音）
超过8个时：按优先级排序，低优先级SFX跳过
高优先级：基地扣血、Boss、技能、胜负
中优先级：攻击命中、金币、道具
低优先级：敌人死亡（大批量死亡时降级）
```

**BGM音量淡变**
```
淡出：volume(t) = start_volume × (1 - t/fade_duration)
淡入：volume(t) = target_volume × (t/fade_duration)
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 同帧大量敌人死亡音效 | 超过8个SFX并发上限，敌人死亡音效（低优先级）随机跳过 |
| BGM切换时旧BGM未播完 | 立即开始淡出，不等待当前循环结束 |
| 音频文件缺失 | 静默跳过（不崩溃），写入警告日志 |
| 主音量设为0 | 所有音频静音，BGM继续计时（位置不停止）以便恢复音量后同步 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| SFX并发上限 | 8个 | 4~16个 | 越高音效越完整，越低防爆音效果越好 |
| BGM淡变时长 | 1.0秒 | 0.3~2.0秒 | 越短切换越突兀，越长过渡越平滑 |
| 默认BGM音量 | 60 | 40~80 | 越低BGM越不干扰SFX感知 |
| 总音频资源大小 | <50MB | <100MB | 影响游戏包体大小 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 动物攻击命中时播放对应SFX | 猎豹攻击敌人，确认有打击音效播放 |
| AC-2 | Build阶段播放轻快BGM，波次开始时切换为战斗BGM | 开始波次时确认BGM发生变化 |
| AC-3 | 基地扣血时播放低沉警报音 | 让敌人到达终点，确认警报音效播放 |
| AC-4 | 音量调节立即生效 | 在设置中调低SFX音量，确认下次攻击音效音量降低 |
| AC-5 | 音量设置在重启后保持 | 调整音量后重启游戏，确认音量设置保留 |
| AC-6 | 同帧大量死亡不导致音频爆音 | 清场10+敌人，确认音频不出现爆音或卡顿 |
