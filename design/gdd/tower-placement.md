# 动物塔放置系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 塔防基础（玩家建造行为的核心交互）

---

## Overview（概述）

动物塔放置系统处理玩家将动物放置到地图上的完整流程：从选择动物、拖拽预览、合法性校验、金币扣除，到最终落地生成塔单位。系统还处理塔的移除（回收部分金币）以及三角龙冲撞触发的塔位移逻辑。放置合法性依赖碰撞检测系统判断格子是否可用，金币扣除依赖资源/货币系统。

---

## Player Fantasy（玩家幻想）

放置动物是塔防游戏最核心的「建造爽感」。拖着一只猎豹到合适位置、松手看它就位开始攻击——这个动作要干净、流畅、有反馈。预览阶段的绿色/红色格子告诉玩家「能放/不能放」，不需要任何说明。三角龙冲撞推开塔的瞬间应该有冲击感，塔位移后玩家重新调整布阵的过程是紧张刺激的战术反应。

## Detailed Design（详细设计）

### 放置流程

```
1. 玩家从UI动物栏选择动物 → 进入DRAGGING状态
2. 鼠标移动时显示预览塔（半透明）：
   - 目标格子合法 → 预览绿色
   - 目标格子非法 → 预览红色
3. 玩家松开鼠标（放置确认）：
   - 合法 + 金币足够 → 扣除金币，生成塔，进入IDLE
   - 合法 + 金币不足 → 显示「金币不足」提示，取消放置
   - 非法 → 取消放置，返回动物栏
4. 玩家按ESC或右键 → 取消放置，返回IDLE
```

### 放置合法性规则

| 条件 | 合法 | 说明 |
|------|------|------|
| 目标格子为普通地面 | ✅ | 标准放置 |
| 目标格子已有其他塔 | ❌ | 格子被占用 |
| 目标格子为水域 | ❌ | 不可在水上建塔 |
| 目标格子为地形边界 | ❌ | 超出地图范围 |
| 目标格子为敌人路径 | ❌ | 路径格子不可放塔（关卡设计保证） |
| 目标格子为陷阱格 | ❌ | 陷阱已占用该格 |
| 金币余额 ≥ 动物cost | ✅ | 经济条件满足 |

> **设计约定**：敌人路径格子由关卡/地图系统标记为不可放置区域，碰撞检测系统读取该标记。

### 塔移除（主动回收）

```
触发：玩家选中已放置的塔，点击「回收」按钮
效果：
  - 移除塔单位
  - 返还金币 = floor(AnimalDB.get(id).cost × sell_ratio)
  - sell_ratio = 0.5（回收50%）
  - 通知寻路系统该格子已释放
```

### 三角龙冲撞位移

```
触发：三角龙charge能力命中路径上的塔
效果：
  1. 计算推开方向（三角龙移动方向的法线）
  2. 目标格子 = 当前格子 + 推开方向（1格）
  3. 目标格子合法（空地）→ 塔移动到目标格子
  4. 目标格子非法（已有塔/边界）→ 塔被摧毁，无金币返还
  5. 通知寻路系统地图状态改变，触发局部路径重算
```

### 状态机

| 状态 | 说明 |
|------|------|
| IDLE | 无放置操作进行中 |
| DRAGGING | 玩家正在拖拽动物，显示预览 |
| PLACED | 塔已放置，等待下一操作 |

## Formulas（公式）

**回收金币**
```
回收金币 = floor(AnimalDB.get(id).cost × 0.5)
示例：猎豹cost=100 → 回收50金币
示例：穿山甲cost=160 → 回收80金币
```

**每格最大塔数**
```
每格最多1个塔单位（硬性限制）
```

**放置成本**
```
放置成本 = AnimalDB.get(id).cost
范围：70（变色龙）~ 180（大象）
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 拖拽中敌人波次开始 | 允许继续拖拽，不强制取消 |
| 放置时金币刚好被其他消耗扣光 | 松手时重新检查余额，不足则取消并提示 |
| 三角龙推开方向目标格已有塔 | 被推塔摧毁（无返还），推开格已有塔则两塔都摧毁 |
| 三角龙推开至地图边界外 | 塔摧毁（无返还） |
| 快速连续点击放置同一格子 | 第一次放置成功后格子已被占，第二次判定非法，自动拒绝 |
| 塔在攻击动画中被回收 | 立即中断攻击动画，移除塔单位 |
| 回收金币导致余额超出上限 | 金币无上限，直接加入余额 |
| 放置时路径被完全封堵 | 碰撞检测系统标记该格为敌人路径不可放，理论上不应发生；若发生则拒绝放置 |

## Dependencies（依赖关系）

### 上游依赖

| 系统 | 使用内容 |
|------|----------|
| 碰撞检测系统 | 判断目标格子是否可放置（碰撞体查询） |
| 资源/货币系统 | 放置时调用spend()，回收时调用earn() |
| 动物数据库 | AnimalDB.get(id).cost 提供放置成本和回收计算基础 |
| 关卡/地图系统 | 提供敌人路径格子标记（不可放置区域） |

### 下游依赖

| 系统 | 使用内容 | 状态 |
|------|---------|------|
| 寻路系统 | 塔放置/移除/位移后通知地图状态改变，触发重算 | ✅ 已完成 |
| 动物塔攻击系统 | 塔放置后初始化攻击单位 | ⬜ 未设计 |
| 游戏UI系统 | 放置预览、金币不足提示、回收按钮 | ⬜ 未设计 |
| 敌人AI系统 | 三角龙charge能力触发位移，通知本系统执行 | ⬜ 未设计 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| sell_ratio（回收比例） | 0.5 | 0.3~0.7 | 越高玩家越敢试错重放；越低放置决策压力越大 |
| 三角龙推开距离 | 1格 | 1~2格 | 2格推开破坏性更强，但视觉和寻路处理更复杂 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 拖拽动物时合法格子显示绿色预览，非法格子显示红色 | 拖拽动物到空地（绿）、已有塔的格子（红）、水域（红），确认颜色正确 |
| AC-2 | 放置成功后金币正确扣除 | current_gold=150时放置cost=100的猎豹，确认current_gold变为50 |
| AC-3 | 金币不足时放置失败并提示 | current_gold=50时放置cost=100的猎豹，确认放置失败、金币不变、显示提示 |
| AC-4 | 已有塔的格子不可再放置 | 尝试在已有塔的格子放置新塔，确认被拒绝 |
| AC-5 | 回收塔返还50%金币 | 回收cost=100的猎豹，确认获得50金币 |
| AC-6 | 三角龙冲撞将塔推开1格 | 三角龙冲撞路径上的塔，确认塔移动到相邻格子 |
| AC-7 | 三角龙推向边界时塔被摧毁无返还 | 将塔放置在地图边界旁，三角龙冲撞推向边界，确认塔消失且金币不增加 |
| AC-8 | 塔放置后寻路系统收到地图更新通知 | 放置塔后，检查PathfindingSystem收到obstacle_changed事件 |
