# 动物数据库

> **状态**: 设计中
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 支柱二（Build 进化自由）、支柱三（纪元对抗叙事）

---

## Overview（概述）

动物数据库是 TiggyTD 所有现代动物单位的属性定义中心。它以数据表的形式存储12只初始动物的全部基础属性——包括基础攻击力、攻速、射程、HP、护甲率、移动速度、职业类型标签和技能树根节点引用。所有依赖动物属性的系统（塔放置、塔攻击、技能树、图鉴收集）都从本数据库读取，不允许在其他系统中硬编码动物属性值。数据库本身不包含逻辑，只负责存储和提供数据。

---

## Player Fantasy（玩家幻想）

**玩家幻想**：「每只动物都是独一无二的存在，我能一眼看出它的个性。」

玩家在选择放置哪只动物时，不需要查阅数值表——犀牛看起来就该很抗揍，猎豹看起来就该很快，鹰看起来就该射得很远。数据库中的属性设计要服务于这种直觉：**视觉形象与数值定位必须一致**。

当玩家第一次解锁一只新动物时，他应该能在3秒内猜出它的大致职责。当他投入技能树后，他应该感受到数值在支撑他的选择——「我选了猎豹连击流，它的攻速真的比其他动物快很多」。

数据库设计原则：
1. **差异化**：12只动物的属性组合不能雷同，每只都有1-2个极端化的属性
2. **直觉一致**：属性数值要匹配动物的视觉形象和生物直觉
3. **技能树空间**：基础属性要留出技能树改造的成长空间（不能初始就满配）

## Detailed Design（详细设计）

### Core Rules（核心规则）

动物数据库以静态数据表形式存储，运行时只读，所有系统通过动物ID查询属性，不得直接修改数据库条目。

#### 属性字段说明

| 字段 | 单位 | 说明 |
|------|------|------|
| `base_atk` | 数值 | 基础攻击力，对应伤害计算系统范围 5~200 |
| `atk_speed` | 次/秒 | 每秒攻击次数 |
| `range` | 格 | 攻击射程（地图格为单位） |
| `hp` | 数值 | 塔的生命值（被敌人反击时扣除） |
| `armor` | % | 基础护甲率，减少受到的伤害 |
| `move_speed` | 格/秒 | 可移动塔重新部署时的移动速度 |
| `cost` | 金币 | 首次放置费用 |
| `class` | 标签 | Tank / DPS / Support / Wildcard |

#### Tank 类

| 动物 | base_atk | atk_speed | range | hp | armor | move_speed | cost | class |
|------|----------|-----------|-------|----|-------|------------|------|-------|
| 犀牛 | 45 | 0.8 | 1.5 | 400 | 35% | 1.5 | 120 | Tank |
| 河马 | 30 | 0.6 | 1.2 | 600 | 20% | 1.0 | 150 | Tank |
| 大象 | 55 | 0.5 | 2.0 | 500 | 25% | 1.2 | 180 | Tank |

#### DPS 类

|------|----------|-----------|-------|----|-------|------------|------|-------|

| 鹰 | 50 | 1.0 | 7.0 | 150 | 0% | 2.0 | 130 | DPS |
| 狼群 | 15×3 | 1.5 | 2.5 | 120×3 | 10% | 2.5 | 110 | DPS |

|------|----------|-----------|-------|----|-------|------------|------|-------|

| 水獭 | 10 | 1.5 | 3.0 | 200 | 15% | 2.5 | 80 | Support |
| 孔雀 | 15 | 0.8 | 3.5 | 180 | 10% | 2.0 | 100 | Support |

#### Wildcard 类

| 动物 | base_atk | atk_speed | range | hp | armor | move_speed | cost | class |
|------|----------|-----------|-------|----|-------|------------|------|-------|
| 变色龙 | 5 | 0.5 | 1.0 | 140 | 5% | 3.0 | 70 | Wildcard |
| 蜜獾 | 35 | 1.2 | 1.5 | 300 | 0% | 2.0 | 140 | Wildcard |
| 穿山甲 | 40 | 0.7 | 1.5 | 350 | 40% | 1.5 | 160 | Wildcard |

#### 设计注解

- **狼群**：以「一组3只」为单位，`base_atk` 和 `hp` 是单只数值，实际部署后3只同时存在
- **变色龙**：`base_atk` 极低（5），核心输出来自陷阱触发（基础陷阱伤害=80，由伤害计算系统陷阱公式计算）
- **蜜獾**：`armor=0%` 是设计意图——靠「无视护甲」和「低血量狂暴」而非自身防御生存
- **穿山甲**：最高护甲率（40%），卷甲状态下护甲率临时提升至70%
- **鹰**：射程最远（7格），全场唯一能越过障碍直接攻击的单位

### States and Transitions（状态与转换）

数据库为静态只读结构，无状态转换。运行时流程：

1. 游戏启动时从配置文件加载全部动物数据到内存
2. 各系统通过 `AnimalDB.get(animal_id)` 查询，返回只读数据对象
3. 技能树系统在运行时计算动态属性（基础属性 + 技能树加成），不修改数据库

### Interactions with Other Systems（与其他系统的交互）

| 系统 | 交互方式 |
|------|----------|
| 动物塔放置系统 | 查询 `cost`、`class`、`hp` 用于放置验证和初始化 |
| 动物塔攻击系统 | 查询 `base_atk`、`atk_speed`、`range` 用于攻击计算 |
| 伤害计算系统 | 接收 `base_atk` 作为基础伤害输入 |
| 动物技能树系统 | 查询基础属性作为技能树加成的计算基准 |
| 图鉴收集系统 | 查询全部动物列表用于图鉴展示和解锁状态 |
| 游戏 UI 系统 | 查询动物名称、类型标签、属性用于塔选择面板显示 |

## Formulas（公式）

动物数据库本身不包含计算公式，只提供原始数值。以下是下游系统使用本数据库数值的关键公式引用：

**塔放置费用验证**
```
可放置 = 玩家当前金币 >= AnimalDB.get(id).cost
```

**基础伤害输入（来自伤害计算系统）**
```
基础伤害 = AnimalDB.get(id).base_atk × (1 + 技能树攻击加成) + 道具附加伤害
```

**攻击间隔**
```
攻击间隔（秒）= 1.0 / AnimalDB.get(id).atk_speed
实际攻击间隔 = 攻击间隔 × (1 - 技能树攻速加成)  // 技能树攻速加成最高0.75
```

**狼群特殊处理**
```
狼群实际总HP = AnimalDB.get("wolf_pack").hp × 3
狼群实际总DPS = AnimalDB.get("wolf_pack").base_atk × AnimalDB.get("wolf_pack").atk_speed × 3
```

**穿山甲卷甲护甲率**
```
卷甲护甲率 = min(0.75, AnimalDB.get("pangolin").armor + 0.30)  // +30%，上限75%
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 查询不存在的 animal_id | 返回 null，调用方负责处理；不抛出异常，记录警告日志 |
| 狼群3只中有1只死亡 | 剩余2只继续战斗，DPS和HP按实际存活数量计算；全部死亡才算塔被摧毁 |
| 穿山甲卷甲状态下被强制移动 | 卷甲状态解除，恢复基础护甲率40%，移动完成后可重新卷甲 |
| 变色龙陷阱被多只敌人同时踩中 | 每只各自触发一次，陷阱触发后进入冷却（冷却时间由技能树定义，基础3秒） |
| 蜜獾HP降至0后触发「不服输」回血 | 回满20% HP后短暂无敌0.5秒，每局最多触发3次（防止无限续命） |
| 游戏启动时配置文件缺失或损坏 | 游戏无法启动，显示错误提示「动物数据加载失败，请重新安装」 |
| 技能树加成导致atk_speed超过合理范围 | atk_speed 运行时上限为 8.0 次/秒（由动物塔攻击系统负责截断） |

## Dependencies（依赖关系）

### 上游依赖

无。动物数据库是 Layer 0 系统，不依赖任何其他游戏系统。数据来源为设计时编写的静态配置文件（JSON 或 Godot Resource 格式）。

### 下游依赖（这些系统依赖本数据库）

| 系统 | 使用字段 | 状态 |
|------|---------|------|
| 动物塔放置系统 | `cost`、`hp`、`class`、`move_speed` | ⬜ 未设计 |
| 动物塔攻击系统 | `base_atk`、`atk_speed`、`range` | ⬜ 未设计 |
| 伤害计算系统 | `base_atk` | ✅ 已完成 |
| 动物技能树系统 | 全部字段作为加成基准 | ⬜ 未设计 |
| 协同效果系统 | `class` 标签用于协同触发条件判断 | ⬜ 未设计 |
| 图鉴收集系统 | 全部动物列表、名称、类型 | ⬜ 未设计 |
| 游戏 UI 系统 | 名称、`class`、所有属性用于面板显示 | ⬜ 未设计 |

### 数据接口约定

```
AnimalDB.get(animal_id: String) -> AnimalData
AnimalDB.get_all() -> Array[AnimalData]
AnimalDB.get_by_class(class: String) -> Array[AnimalData]

AnimalData {
  id: String          // 唯一标识，如 "rhino"、"wolf_pack"
  name_key: String    // 本地化键名，如 "animal.rhino.name"
  class: String       // "Tank" | "DPS" | "Support" | "Wildcard"
  base_atk: float
  atk_speed: float
  range: float
  hp: int
  armor: float        // 0.0 ~ 0.75
  move_speed: float
  cost: int
  skill_tree_root: String  // 技能树根节点ID引用
}
```

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| Tank类平均HP | 500 | 350~700 | 低于350前排太脆，敌人轻易突破；高于700后期清场太慢 |
| DPS类平均base_atk | 30 | 20~50 | 影响前中期清怪速度，过高导致Build价值感下降 |
| Support类cost均值 | 90 | 70~120 | 过贵导致玩家不愿投资辅助，破坏职业协同体验 |
| 猎豹atk_speed | 2.5 | 2.0~3.5 | 低于2.0连击感消失；高于3.5特效过于密集影响画面清晰度 |
| 鹰range | 7.0 | 5.0~9.0 | 低于5.0失去「全图狙击」差异化；高于9.0平衡性崩坏 |
| 穿山甲卷甲护甲加成 | +30% | +20%~+40% | 影响穿山甲要塞流的生存极限 |
| 蜜獾「不服输」触发上限 | 3次/局 | 2~5次 | 低于2次存在感太弱；高于5次近乎无敌破坏平衡 |
| 变色龙陷阱基础伤害 | 80 | 60~120 | 影响陷阱流Build相对于DPS塔的竞争力 |
| 变色龙陷阱冷却 | 3秒 | 2~5秒 | 低于2秒陷阱覆盖率过高；高于5秒陷阱流体验太差 |
| 狼群单只HP | 120 | 80~180 | 影响狼群在敌人反击下的生存时间 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 所有12只动物数据可正常加载，无缺失字段 | 游戏启动后调用 `AnimalDB.get_all()`，确认返回12条完整记录 |
| AC-2 | 每只动物的 `base_atk` 在伤害计算系统范围 5~200 内 | 遍历全部记录，断言 `5 <= base_atk <= 200` |
| AC-3 | 每只动物的 `armor` 在 0.0~0.75 范围内 | 遍历全部记录，断言 `0.0 <= armor <= 0.75` |
| AC-4 | 狼群以3只为单位正确初始化 | 放置狼群后，场上出现3个独立单位，各自HP=120 |
| AC-5 | 穿山甲卷甲后护甲率正确变为70% | 触发卷甲，查询当前护甲率，断言 = 0.70 |
| AC-6 | 蜜獾「不服输」每局最多触发3次 | 将蜜獾HP反复降至0，第4次应不再触发回血 |
| AC-7 | 变色龙陷阱触发后进入3秒冷却 | 两只敌人先后踩陷阱，第二只踩时若在3秒内应不触发伤害 |
| AC-8 | 查询不存在的ID返回null不崩溃 | 调用 `AnimalDB.get("nonexistent")`，游戏不崩溃，返回null |
| AC-9 | 所有动物 `cost` 与 `class` 对应关系符合设计 | Tank均值≥120，Support均值≤100，Wildcard在70~160之间 |
| AC-10 | 本地化键名格式正确 | 所有 `name_key` 符合 `animal.[id].name` 格式，本地化系统可正常解析 |
