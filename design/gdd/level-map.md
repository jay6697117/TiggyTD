# 关卡/地图系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27

---

## Overview（概述）

关卡/地图系统定义MVP阶段的单张地图结构：网格布局、敌人行进路径、可放置区域、基地位置。地图是所有其他系统的空间容器——碰撞检测依赖网格坐标、寻路系统在路径上运行、塔放置系统在可放格上操作。MVP只需一张固定地图，手工设计，无程序生成。

---

## Player Fantasy（玩家幻想）

地图是战略博弈的舞台。玩家一眼看到路径的弯道、瓶颈口、宽阔区域，本能地开始规划「在这个弯道堆减速塔、在这个直道放输出塔」——这种空间阅读感是塔防游戏最原始的乐趣。MVP地图要让第一次玩的玩家在30秒内看懂，但在10局内仍有新的阵型发现。

## Detailed Design（详细设计）

### 地图规格

```
地图尺寸：20格（宽）× 15格（高）
格子大小：64×64像素
总像素尺寸：1280×960
坐标系：(0,0) 在左上角，x向右，y向下
```

### 格子类型

| 类型ID | 名称 | 描述 | 可放塔 |
|--------|------|------|--------|
| `PATH` | 路径格 | 敌人行进路线 | 否 |
| `BUILD` | 建造格 | 可放置动物塔 | 是 |
| `BLOCKED` | 封锁格 | 装饰/地形，不可通行不可建造 | 否 |
| `SPAWN` | 刷怪点 | 敌人出生位置（路径起点） | 否 |
| `BASE` | 基地格 | 玩家基地（路径终点） | 否 |

### MVP地图布局（文字描述）

```
路径形状：S形双弯道

刷怪点：左上角 (0, 2)
基地：右下角 (19, 12)

路径走向：
  (0,2) → 向右至 (7,2)           // 上方横道
  (7,2) → 向下至 (7,9)           // 第一竖道
  (7,9) → 向右至 (13,9)          // 中间横道
  (13,9) → 向下至 (13,12)        // 第二竖道（短）
  (13,12) → 向右至 (19,12)       // 下方横道至基地

关键节点：
  弯道1：(7,2)~(7,9) 竖道顶部 — 理想减速/AOE位置
  瓶颈口：(7,9)~(13,9) 中段横道 — 最长直道，适合高射速塔
  弯道2：(13,9)~(13,12) — 第二个减速布防点
```

### 可建造区域规则

```
路径格及其相邻1格（路径边缘格）：不可建造
其余格子：全部为 BUILD 格
装饰封锁格：地图四角各有2×2的 BLOCKED 区域（模拟丛林遮挡）
```

### 地图数据结构

```
MapGrid {
  width: 20
  height: 15
  cells: Cell[20][15]
}

Cell {
  x: int
  y: int
  type: PATH | BUILD | BLOCKED | SPAWN | BASE
  occupied_by: tower_id | null  // 当前放置的塔（BUILD格）
}

Path {
  waypoints: [(0,2), (7,2), (7,9), (13,9), (13,12), (19,12)]
  // 寻路系统使用waypoints做A*引导
}
```

### 地图加载流程

```
游戏启动 → load_map("mvp_map_01"):
1. 读取地图数据（硬编码或JSON文件）
2. 初始化 MapGrid，设置每格 type
3. 向寻路系统注册 Path.waypoints
4. 向碰撞检测系统注册 BLOCKED/PATH 格为不可通行
5. 渲染地图贴图（路径/草地/装饰层）
6. 标记 SPAWN 格，初始化敌人出生点
7. 标记 BASE 格，注册基地碰撞触发器
```

### 视觉层次

```
Layer 0（底层）：草地地板贴图（全图覆盖）
Layer 1（路径层）：泥土/石板路贴图（覆盖 PATH 格）
Layer 2（装饰层）：树木/岩石精灵（BLOCKED 格）
Layer 3（塔层）：动物塔精灵（BUILD 格上方）
Layer 4（敌人层）：敌人精灵（PATH 格上方）
Layer 5（UI层）：放置预览、格子高亮
```

## Formulas（公式）

**世界坐标 ↔ 格子坐标转换**
```
grid_x = floor(world_x / 64)
grid_y = floor(world_y / 64)

world_x = grid_x × 64 + 32  // 格子中心x
world_y = grid_y × 64 + 32  // 格子中心y
```

**路径总长度（格数）**
```
段1：(0,2)→(7,2) = 7格
段2：(7,2)→(7,9) = 7格
段3：(7,9)→(13,9) = 6格
段4：(13,9)→(13,12) = 3格
段5：(13,12)→(19,12) = 6格
总路径长度 = 29格 = 29×64 = 1856像素
```

**敌人穿越时间估算**
```
迅猛龙(speed=3.0格/秒)：29/3.0 ≈ 9.7秒（无减速）
腕龙(speed=1.0格/秒)：29/1.0 = 29秒（无减速）
```

**可建造格子数量**
```
总格数：20×15 = 300
PATH格：约29格（路径格）
路径边缘格：约58格（路径两侧各1格）
BLOCKED格：4角 × 4格 = 16格
可建造格约：300 - 29 - 58 - 16 ≈ 197格
（实际需要美术确认边缘格的精确范围）
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 玩家尝试在 PATH 格放塔 | 塔放置系统检查 cell.type != PATH，拒绝放置并提示「此处不可建造」 |
| 玩家尝试在已有塔的格子放塔 | 检查 cell.occupied_by != null，拒绝放置 |
| 三角龙将塔推入 PATH 格 | 推动后检查目标格 type，若为 PATH 则推动失败（塔不移动）；参见碰撞检测系统 |
| 地图加载失败（文件缺失） | MVP阶段硬编码地图数据，不依赖外部文件，此情况不存在 |
| 敌人在路径拐角处的精确位置 | 寻路系统负责处理waypoint转向；地图系统只提供waypoints坐标 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 地图尺寸 | 20×15 | 15×10~25×20 | 越大可建造空间越多，越小决策更集中 |
| 路径总长度 | 29格 | 20~40格 | 越长敌人暴露在攻击范围内时间越长，越容易被消灭 |
| 弯道数量 | 2个 | 1~4个 | 越多战略分布点越多，布防灵活性更高 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 地图正确渲染20×15网格，路径可见 | 启动游戏，确认地图显示S形路径，草地和路径贴图正确 |
| AC-2 | 敌人从SPAWN点出生并沿路径行进至BASE | 启动波次，确认敌人从左上角出现，沿S形路径到达右下角 |
| AC-3 | PATH格不可放置动物塔 | 点击任意路径格尝试放塔，确认显示「不可建造」提示 |
| AC-4 | BUILD格可正常放置动物塔 | 点击路径旁的草地格，确认可放置动物塔 |
| AC-5 | 世界坐标与格子坐标转换正确 | 点击像素位置(448,160)，确认对应格子(7,2)被高亮 |
| AC-6 | 路径waypoints正确传递给寻路系统 | 确认敌人在(7,2)和(7,9)处正确转向，不走直线穿越草地 |
