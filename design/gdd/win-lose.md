# 胜负判定系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27

---

## Overview（概述）

胜负判定系统监控两个核心变量：基地HP（敌人穿越终点时扣除）和当前波次进度。当基地HP归零时触发失败流程；当最终波次（第10波）所有敌人被消灭时触发胜利流程。系统是游戏循环的终点，连接波次系统、敌人AI系统和游戏UI系统，确保每局有明确的赢和输。

---

## Player Fantasy（玩家幻想）

胜利的瞬间是整局积累的情绪释放——第10波Boss倒下、基地HP还剩最后一格，那种惊险卡线的满足感是Roguelite游戏的精髓。失败不是惩罚而是复盘起点：基地HP从100扣到0的过程是清晰的失败叙事，玩家能看懂哪一波打穿了防线、哪个位置出了问题，为下一局的Build调整提供方向。

## Detailed Design（详细设计）

### 核心变量

```
base_hp: int          // 基地当前HP，初始值=10（MVP阶段）
base_hp_max: int      // 基地最大HP=10
current_wave: int     // 当前波次（1~10）
total_waves: int      // MVP总波次=10
game_state: enum      // PLAYING / WIN / LOSE
```

### 失败条件

```
触发时机：敌人单位到达路径终点（终点格）
扣血逻辑：
  base_hp -= enemy.base_damage_to_base  // 默认每只敌人扣1点
  enemy.despawn()                        // 敌人消失（不继续在场）
  if base_hp <= 0:
    base_hp = 0
    trigger_lose()
```

### 胜利条件

```
触发时机：第10波次结束事件触发后
检测逻辑（在波次系统的 on_wave_complete 回调中）：
  if current_wave == total_waves and alive_enemies.count == 0:
    trigger_win()
```

### 胜利流程

```
trigger_win():
1. game_state = WIN
2. 停止所有敌人生成和移动（冻结游戏世界）
3. 播放胜利特效（金色粒子、英雄胜利动画）
4. 显示胜利UI面板：
   - "胜利！" 标题
   - 存活基地HP：base_hp / base_hp_max
   - 本局统计：击杀数、最高连击波次
   - 按钮：「再来一局」/ 「返回主菜单」
5. 保存通关记录（调用存档系统）
```

### 失败流程

```
trigger_lose():
1. game_state = LOSE
2. 停止所有敌人生成（已在场敌人继续动画但冻结逻辑）
3. 播放失败特效（基地爆炸动画、屏幕变暗）
4. 显示失败UI面板：
   - "失败" 标题
   - 撑到第几波：current_wave / total_waves
   - 击杀数统计
   - 按钮：「再来一局」/ 「返回主菜单」
5. 不保存通关记录
```

### 基地HP显示

```
基地HP始终显示在UI顶部
每次扣血：数字变红闪烁0.5秒，屏幕轻微震动（shake_intensity=5px, duration=0.3秒）
基地HP=1时：持续红色闪烁警告
```

## Formulas（公式）

**基地扣血**
```
base_hp -= enemy.base_damage_to_base
默认 base_damage_to_base = 1（所有普通敌人）
Boss base_damage_to_base = 3（Boss穿越直接扣3点）
MVP base_hp_max = 10
```

**胜利判定**
```
win_condition = (current_wave == 10) AND (alive_enemies.count == 0)
```

**失败判定**
```
lose_condition = (base_hp <= 0)
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 最后一波最后一只敌人同时到达终点和被击杀 | 死亡事件优先于到达终点事件；若先到达终点再死亡则扣血并检查胜利 |
| 多只敌人同帧到达终点，base_hp恰好归零 | 全部扣血后检查一次 lose_condition，触发一次失败流程 |
| 第10波结束但仍有敌人存活（如被推出路径） | alive_enemies.count > 0，不触发胜利；继续等待所有敌人死亡或到达终点 |
| 玩家在波次间隙（build阶段）基地HP已0 | 理论上不可能（只有敌人到达终点才扣血，build阶段无敌人在路径上） |
| 胜负UI弹出期间玩家仍可操作英雄 | 触发胜负后立即冻结输入，防止UI弹出后继续操作 |
| Boss扣3点导致base_hp从2变为-1 | base_hp = max(0, base_hp - damage)，显示0，触发失败 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 基地最大HP | 10 | 5~20 | 越少容错越低，玩家压力越大；越多允许更多敌人漏过 |
| 普通敌人扣血 | 1点 | 1~2点 | 若为2点则基地只能承受5只敌人漏过 |
| Boss扣血 | 3点 | 2~5点 | Boss穿越的严重程度 |
| 屏幕震动强度 | 5px | 0~15px | 扣血反馈的冲击感 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 敌人到达终点扣1点基地HP | 让一只迅猛龙跑到终点，确认基地HP从10变为9 |
| AC-2 | base_hp归零触发失败流程 | 让10只敌人到达终点，确认失败UI弹出、游戏冻结 |
| AC-3 | 第10波所有敌人消灭触发胜利 | 手动设置wave=10，消灭最后一只敌人，确认胜利UI弹出 |
| AC-4 | 胜利后无法继续操作英雄 | 胜利UI弹出后点击地图，确认英雄不移动 |
| AC-5 | 扣血时屏幕震动且HP数字红色闪烁 | 敌人到达终点，确认震动效果和UI闪烁持续约0.5秒 |
| AC-6 | Boss扣3点HP | Boss到达终点，确认基地HP减少3（而非1） |
| AC-7 | 「再来一局」按钮重置游戏状态 | 点击再来一局，确认base_hp=10、wave=1、地图清空 |
