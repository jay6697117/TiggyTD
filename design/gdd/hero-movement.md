# 英雄移动系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 支柱一（图腾指挥感——英雄是玩家在战场上的化身）

---

## Overview（概述）

英雄移动系统驱动玩家控制的老虎英雄在地图上移动。玩家点击地图任意可通行位置，英雄沿寻路系统计算的A*路径移动至目标点。英雄可穿越动物塔占据的格子，不可进入水域和地形边界。本系统负责输入处理（点击检测）、移动执行（路径点跟随）、动画状态切换，不负责路径计算（由寻路系统负责）和碰撞体定义（由碰撞检测系统负责）。

---

## Player Fantasy（玩家幻想）

英雄老虎是玩家在战场上的「身体」。点击目标、老虎流畅奔跑过去——这个操作必须感觉灵敏、有重量、有生命感。玩家会频繁移动英雄去释放技能、触发光环、支援薄弱防线。如果移动感觉迟钝或卡顿，玩家会感到沮丧。英雄在动物塔之间穿梭、快速到位的「战场机动感」，是区别于普通塔防的核心体验之一。

## Detailed Design（详细设计）

### 输入处理

```
触发方式：玩家鼠标左键点击地图（非UI区域）
点击目标：鼠标点击位置转换为地图格子坐标
无效点击：
  - 点击水域 → 忽略（英雄停在当前位置）
  - 点击地形边界外 → 忽略
  - 点击UI层 → UI系统消费，不传递给移动系统
连续点击：立即丢弃当前路径，以新目标重新寻路
```

### 移动执行

```
1. 收到目标坐标 → 调用 PathfindingSystem.find_path(hero_pos, target_pos)
2. 获得路径点数组 waypoints[]
3. 按顺序移动至每个路径点：
   每帧移动量 = hero_speed × delta
   方向 = normalize(waypoints[current_index] - hero_pos)
   hero_pos += 方向 × 每帧移动量
4. 到达路径点（距离 < arrival_threshold）→ current_index++
5. 到达最终路径点 → 进入IDLE状态，停止移动
```

### 状态机

| 状态 | 说明 | 转换条件 |
|------|------|----------|
| IDLE | 英雄静止，等待输入 | 收到有效点击 → MOVING |
| MOVING | 英雄沿路径移动 | 到达终点 → IDLE；收到新点击 → 重新计算路径，保持MOVING |
| CASTING | 英雄释放技能，移动暂停 | 技能系统通知技能结束 → IDLE或MOVING |

### 移动属性（来自动物数据库）

```
hero_speed = AnimalDB.get("tiger").move_speed = 3.5 格/秒
arrival_threshold = 0.1 格（到达判定距离）
碰撞体：圆形，半径0.4格（来自碰撞检测系统）
```

### 可通行规则（来自寻路系统）

| 格子类型 | 英雄可通行 |
|---------|----------|
| 普通地面 | ✅ |
| 水域 | ❌ |
| 地形边界 | ❌ |
| 塔占据格 | ✅（不同碰撞层） |
| 陷阱格 | ✅ |

## Formulas（公式）

**每帧移动**
```
每帧移动量 = hero_speed × delta
hero_speed = 3.5（格/秒）
delta = 帧间隔时间（秒），60fps时约0.0167秒
每帧移动量示例：3.5 × 0.0167 ≈ 0.058格/帧
```

**到达判定**
```
距离 = (waypoints[i] - hero_pos).length()
若 距离 < arrival_threshold（0.1格）→ 视为到达，移向下一路径点
```

**移动方向**
```
方向向量 = normalize(target_waypoint - hero_pos)
新位置 = hero_pos + 方向向量 × (hero_speed × delta)
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 点击水域 | 忽略输入，英雄保持当前状态 |
| 点击地形边界外 | 忽略输入 |
| 移动中收到新点击 | 立即重新寻路，丢弃旧路径，不等当前路径点到达 |
| 寻路系统返回空路径（目标不可达） | 英雄保持当前位置，进入IDLE |
| 英雄到达目标时目标格子变为障碍 | 英雄停在最近可通行格子（寻路系统已处理） |
| CASTING状态收到点击 | 记录目标位置，技能结束后立即开始移动 |
| 英雄speed被减速效果修改 | 直接读取修改后的speed值，不需要特殊处理 |
| 路径点间距极大（长直线） | 每帧按delta插值，不会跳过路径点 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| hero_speed | 3.5格/秒 | 2.5~5.0 | 越快操控感越强但越难被拦截；越慢战术定位感越强 |
| arrival_threshold | 0.1格 | 0.05~0.2 | 太小容易在路径点附近抖动；太大路径精度下降 |
| 减速率上限（来自随机性系统） | 80% | — | 英雄最慢speed = 3.5×0.2 = 0.7格/秒，仍可移动 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 点击可通行位置，英雄沿A*路径移动至目标 | 点击地图任意可通行位置，观察英雄移动路径不穿越障碍 |
| AC-2 | 英雄到达目标后进入IDLE停止移动 | 英雄到达点击位置后静止，不继续移动 |
| AC-3 | 连续点击立即改变目标 | 英雄移动中再次点击新位置，英雄立即转向新目标 |
| AC-4 | 点击水域英雄不移动 | 点击水域格子，英雄保持当前位置 |
| AC-5 | 英雄可穿越动物塔格子 | 英雄路径经过塔所在格子，正常通过不被阻挡 |
| AC-6 | 英雄无法穿越地形边界 | 操控英雄向地图边界移动，确认被阻挡 |
| AC-7 | 移动速度符合设定值 | 计时英雄穿越10格距离，用时约2.86秒（10÷3.5） |
| AC-8 | CASTING状态期间英雄不移动 | 释放技能时点击新位置，英雄技能结束前不移动 |
