# 随机性系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27
> **实现支柱**: 技术基础（支撑商店/波次/掉落的所有随机行为）

---

## Overview（概述）

随机性系统为 TiggyTD 中所有随机行为提供统一的随机数生成服务。系统基于**带种子的伪随机（Seeded RNG）**，支持存档重现。对外提供三种接口：纯随机（范围内均匀分布）、权重随机（按权重表抽取）、保底随机（权重随机 + 连续未出现保底）。商店刷新、波次敌人生成、道具掉落均通过本系统获取随机结果。

---

## Player Fantasy（玩家幻想）

随机性是「每局不同」的根源。玩家在商店看到意外出现的稀有道具时感到惊喜，在连续几轮没出现想要的动物后终于刷出时感到满足——这种张弛感来自保底机制的精确控制。波次的敌人组合每局略有变化，让老玩家也不会完全「背板」。随机性不是惩罚，而是制造期待和意外惊喜的工具。

## Detailed Design（详细设计）

### 随机源设计

**带种子的伪随机（Seeded RNG）**

```
种子生成：游戏开始时取系统时间戳作为主种子
存档时：将当前RNG状态（种子+已消耗次数）一同存入存档
读档时：恢复RNG状态，后续随机结果与存档时完全一致
实现：Godot 内置 RandomNumberGenerator（rng.seed + rng.state）
```

**多个独立RNG实例**（避免一处随机影响另一处）

| 实例名 | 用途 |
|--------|------|
| rng_shop | 商店刷新 |
| rng_wave | 波次敌人生成 |
| rng_drop | 道具掉落 |
| rng_misc | 其他杂项（暴击判定等） |

### 三种随机接口

**接口1：纯随机（Uniform Random）**

```
RandomSystem.uniform(min, max) → float
RandomSystem.uniform_int(min, max) → int
用途：数值微扰（伤害浮动±5%）、坐标随机等
```

**接口2：权重随机（Weighted Random）**

```
RandomSystem.weighted_pick(items: Array[{id, weight}]) → id
算法：
  1. 计算所有权重之和 total_weight
  2. 生成 r = uniform(0, total_weight)
  3. 遍历items，累加权重直到超过r，返回当前item
用途：商店物品池、波次敌人种类、掉落物品种类
```

**接口3：保底随机（Pity Random）**

```
RandomSystem.pity_pick(items, pity_counters) → id
规则：
  - 基础行为同权重随机
  - 每个item维护一个 pity_counter（连续未被选中次数）
  - pity_counter 达到 pity_threshold 时，该item权重临时×10
  - 被选中后该item的pity_counter重置为0
用途：商店稀有动物保底、道具掉落保底
```

### 各系统使用方式

| 使用系统 | 接口 | RNG实例 | 说明 |
|---------|------|---------|------|
| 商店刷新 | 保底随机 | rng_shop | 从动物池/道具池各抽N个 |
| 波次生成 | 权重随机 | rng_wave | 按波次概率表抽取敌人种类和数量 |
| 道具掉落 | 保底随机 | rng_drop | 敌人死亡时概率掉落道具 |
| 伤害浮动 | 纯随机 | rng_misc | 实际伤害 = 基础伤害 × uniform(0.95, 1.05) |
| 暴击判定 | 纯随机 | rng_misc | uniform(0,1) < crit_rate → 暴击 |

## Formulas（公式）

**权重随机抽取**
```
输入：items = [{id:"tiger", weight:10}, {id:"lion", weight:5}, {id:"bear", weight:2}]
total_weight = 10 + 5 + 2 = 17
r = uniform(0, 17)  // 例：r = 12.3
累加：tiger(10) < 12.3，lion(10+5=15) > 12.3 → 返回 "lion"
```

**保底随机权重临时加成**
```
临时权重 = base_weight × (1 + pity_bonus × pity_counter)
pity_bonus = 9.0（使pity_counter=1时权重×10）
// 例：base_weight=2, pity_threshold=5, pity_counter=5
// 临时权重 = 2 × (1 + 9.0 × 1) = 20（当counter≥threshold时触发×10）
```

**伤害浮动**
```
实际伤害 = floor(base_damage × uniform(0.95, 1.05))
变量：base_damage = 伤害计算系统输出的基础伤害值
范围：±5%，向下取整
示例：base_damage=100 → 实际伤害范围 [95, 105]
```

**暴击判定**
```
r = uniform(0.0, 1.0)
若 r < crit_rate → 暴击，伤害×2.0
crit_rate 来源：动物数据库 AnimalDB.get(id).crit_rate，范围 0.0~0.3
示例：crit_rate=0.1，r=0.07 → 暴击
```

**商店保底示例**
```
动物「渡渡鸟」base_weight=3，pity_threshold=5
假设已连续4轮未出现（pity_counter=4），未达阈值，权重仍=3
第5轮未出现（pity_counter=5），达到阈值，临时权重=3×10=30
第5轮必然大概率被抽中（权重×10相当于稀有度提升10倍）
被选中后pity_counter重置为0
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 权重表为空 | 报错并返回null，调用方需检查返回值；关卡设计保证池子不为空 |
| 所有权重为0 | 视为均等权重，每个item权重临时设为1 |
| 单个item的保底counter永远不触发 | 仅当pool中有该item时才计数；item从pool移除时counter清零 |
| 读档后RNG状态恢复但结果不一致 | 存档时同时存 rng.seed 和 rng.state，读档时两者都恢复，否则结果不一致 |
| 多个系统同帧同时请求随机数 | 各系统使用独立RNG实例，不存在竞争；同一实例内按调用顺序串行消耗 |
| 伤害浮动导致0伤害 | base_damage≥1时浮动×0.95≥0，floor后最小为0；实际伤害最小值为1（由伤害计算系统保证） |
| 波次概率表权重之和≠100 | 权重随机算法基于相对权重，不要求之和为100；权重表只需保证各项weight>0 |
| 保底阈值设为0 | 等同于每次都触发保底（权重×10恒生效），视为配置错误；加载时校验pity_threshold≥1 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 伤害浮动范围 | ±5%（0.95~1.05） | ±0%~±15% | 范围越大伤害越「飘」，超过±10%玩家会感到不可控 |
| 暴击伤害倍率 | 2.0× | 1.5~3.0 | 低于1.5暴击无感；高于3.0单次暴击过于决定性 |
| 保底触发阈值 pity_threshold | 5（商店） | 3~8 | 越低越频繁保底，稀有物品出现越快；越高玩家等待越久 |
| 保底权重倍率 pity_bonus | 9.0（效果×10） | 4.0~19.0 | 低于4.0保底效果不明显；高于19.0相当于强制出现 |
| 商店每轮刷新数量 | 3个 | 2~5 | 越多选择越多但界面越杂；越少选择压力越大 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 相同种子产生相同随机序列 | 用同一seed初始化两个RNG实例，连续调用100次uniform，结果完全一致 |
| AC-2 | 存档读档后随机结果可重现 | 记录存档时的rng.state，读档后再生成10个随机数，与存档前结果一致 |
| AC-3 | 权重随机分布符合预期 | 运行10000次weighted_pick，各item出现频率误差<5%（相对权重比） |
| AC-4 | 保底机制在阈值处触发 | 配置pity_threshold=5，连续4次不出现目标item，第5次必然大概率出现 |
| AC-5 | 多个RNG实例互不干扰 | rng_shop消耗100次后，rng_wave的输出与仅消耗0次时相同 |
| AC-6 | 伤害浮动在±5%范围内 | 对base_damage=100运行10000次，实际伤害均在[95,105]内 |
| AC-7 | 权重表为空时返回null且不崩溃 | 传入空数组调用weighted_pick，返回null，无报错崩溃 |
| AC-8 | 保底counter被选中后重置为0 | 触发保底选中后，再次查询该item的pity_counter，值为0 |
