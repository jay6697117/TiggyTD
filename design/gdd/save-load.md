## Overview（概述）

存档/读档系统负责 TiggyTD 所有需要持久化的游戏数据的读写管理。系统分三层：**局内临时存档**（断点续玩，关闭游戏后可继续）、**跨局永久存档**（跨局成长、图鉴解锁、技能树永久进度）、**游戏设置存档**（音量、语言、键位等）。存档格式统一使用 Godot Resource（.tres/.res），与引擎深度集成。支持本地3个存档槽 + Steam 云存档同步备份。本系统也是动物数据库、道具数据库、敌人数据库配置文件的加载入口。

---

## Player Fantasy（玩家幻想）

**玩家幻想**：「我的成长永远不会丢失。」

玩家在关闭游戏时不需要担心进度——局内打到一半可以随时退出，下次打开游戏从断点继续。跨局解锁的动物、技能树永久节点、图鉴记录都安全保存。Steam 云存档让玩家换台电脑也能无缝继续。

存档系统是「无感基础设施」——玩家不应该注意到它的存在，只有在它出问题时才会被察觉。设计目标：存档操作对玩家完全透明，自动触发，零操作负担。

## Detailed Design（详细设计）

### Core Rules（核心规则）

#### 三层存档结构

| 层级 | 名称 | 触发时机 | 数据内容 | 文件路径 |
|------|------|---------|---------|----------|
| Layer 1 | 局内临时存档 | 每波结束自动保存 | 当前局状态（动物塔布局、道具装备、英雄技能树局内进度、当前波次、金币） | `user://saves/slot_{n}/session.tres` |
| Layer 2 | 跨局永久存档 | 一局结束（胜利或失败）后保存 | 图鉴解锁、永久技能树进度、成就、历史最高分 | `user://saves/slot_{n}/progress.tres` |
| Layer 3 | 游戏设置存档 | 设置修改后立即保存 | 音量、语言（zh_CN/en_US）、键位绑定、画质 | `user://settings.tres` |

#### 存档槽规则

- 支持3个独立存档槽（slot_0、slot_1、slot_2）
- 每个槽位独立存储 Layer 1 和 Layer 2 数据
- Layer 3（设置）全局唯一，不区分槽位
- 新建存档时初始化空的 progress.tres，session.tres 不存在（无断点）
- 删除存档槽时同时删除该槽的 session.tres 和 progress.tres

#### 配置文件加载

游戏启动时，本系统负责加载所有静态数据库配置文件：
- `res://data/animals.tres` → 动物数据库
- `res://data/enemies.tres` → 敌人数据库
- `res://data/items.tres` → 道具数据库

配置文件为只读，不写入 `user://` 目录。

#### Steam 云存档

- Layer 2（永久进度）和 Layer 3（设置）同步至 Steam 云
- Layer 1（局内临时）不同步（局内状态与设备绑定）
- 云存档冲突解决策略：取时间戳较新的版本
- Steam SDK 不可用时（非Steam版本）静默跳过云同步

### States and Transitions（状态与转换）

| 状态 | 说明 |
|------|------|
| IDLE | 无存档操作 |
| SAVING | 正在写入文件，此期间不接受新的存档请求（队列等待） |
| LOADING | 正在读取文件 |
| SYNCING | 正在与Steam云同步 |
| ERROR | 存档读写失败，等待错误处理 |

转换规则：
- IDLE → SAVING：触发存档事件
- SAVING → IDLE：写入成功
- SAVING → SYNCING：写入成功且Steam可用（Layer 2/3）
- SYNCING → IDLE：同步完成或超时（10秒）
- 任意状态 → ERROR：IO异常
- ERROR → IDLE：玩家确认错误提示后

### Interactions with Other Systems（与其他系统的交互）

| 系统 | 交互方式 |
|------|----------|
| 动物数据库 | 启动时加载 `animals.tres` 配置文件 |
| 敌人数据库 | 启动时加载 `enemies.tres` 配置文件 |
| 道具数据库 | 启动时加载 `items.tres` 配置文件 |
| 动物技能树系统 | 每波结束时序列化技能树局内状态到 session.tres；一局结束时序列化永久节点到 progress.tres |
| 英雄技能树系统 | 同上 |
| 跨局成长系统 | 读写 progress.tres 中的永久成长数据 |
| 图鉴收集系统 | 读写 progress.tres 中的解锁记录 |
| 本地化系统 | 读取 settings.tres 中的语言设置，游戏启动时最先加载 |
| 游戏UI系统 | 存档槽选择界面、存档操作状态显示 |

## Formulas（公式）

存档系统不包含游戏数值公式。以下是存档操作的技术参数：

**自动保存触发条件**
```
局内自动保存 = 每波结束后（wave_complete 事件触发）
设置自动保存 = 设置变更后延迟2秒（防抖，避免频繁写入）
```

**存档文件大小估算**
```
session.tres 预估：~50KB（12只动物塔状态 + 道具装备 + 波次数据）
progress.tres 预估：~20KB（技能树节点 + 图鉴记录 + 成就）
settings.tres 预估：~5KB
总计：~75KB/存档槽，3槽合计 ~225KB
```

**云同步超时**
```
Steam云同步超时 = 10秒
超时后：本地存档保留，云同步标记为「待同步」，下次启动重试
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 存档写入中途断电/崩溃 | 写入前先写临时文件，写入完成后原子替换正式文件；临时文件存在时下次启动优先检查完整性 |
| 存档文件损坏（校验失败） | 提示「存档损坏，是否重置该存档槽？」，玩家确认后清空；不自动删除 |
| 3个存档槽全部占用时新建 | 不允许超过3个槽，提示「请先删除一个存档」 |
| Steam云存档版本冲突 | 取时间戳较新版本；若本地更新则上传，若云端更新则下载覆盖本地 |
| 磁盘空间不足 | 捕获IO异常，提示「磁盘空间不足，无法保存」；游戏继续运行但不再尝试保存 |
| 配置文件（animals.tres等）缺失 | 游戏无法启动，显示「游戏数据损坏，请重新安装」 |
| 局内临时存档版本与当前游戏版本不兼容 | 提示「存档版本过旧，无法继续，是否放弃该存档？」；永久进度不受影响 |
| 同时触发多个存档请求 | 队列处理，SAVING状态期间新请求入队，逐一执行 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响 |
|------|--------|---------|------|
| 存档槽数量 | 3 | 1~5 | 影响多人共用体验，超过5个UI管理复杂 |
| 设置存档防抖延迟 | 2秒 | 0.5~5秒 | 太短导致频繁IO；太长设置变更有丢失风险 |
| Steam云同步超时 | 10秒 | 5~30秒 | 太短网络差时频繁失败；太长玩家等待感明显 |
| 局内自动保存触发点 | 每波结束 | 每波/每2波/手动 | 更频繁=更安全但IO开销增加 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 局内每波结束自动保存，重启游戏后可从断点继续 | 打到第5波结束后强制关闭游戏，重启后确认从第6波开始 |
| AC-2 | 一局结束后永久进度正确写入progress.tres | 解锁新动物后退出游戏，重启后确认图鉴仍显示已解锁 |
| AC-3 | 3个存档槽各自独立 | 在slot_0和slot_1各进行不同进度，确认两者数据互不影响 |
| AC-4 | 存档损坏时提示而非崩溃 | 手动损坏session.tres文件内容，启动游戏确认出现「存档损坏」提示 |
| AC-5 | 配置文件缺失时游戏显示错误提示而非崩溃 | 删除animals.tres，启动游戏确认显示「游戏数据损坏」提示 |
| AC-6 | 设置修改后2秒内自动保存 | 修改音量后等待3秒，重启游戏确认音量设置保留 |
| AC-7 | 删除存档槽同时清除session和progress | 删除slot_0，确认两个文件均被删除 |
| AC-8 | 存档版本不兼容时提示放弃而非崩溃 | 修改存档版本号后尝试加载，确认出现版本不兼容提示 |
| AC-9 | 磁盘空间不足时游戏继续运行 | 模拟磁盘满，确认游戏继续运行并显示「无法保存」提示 |
| AC-10 | Steam云同步超时后本地存档不受影响 | 断网情况下修改进度，确认本地progress.tres正常写入 |
