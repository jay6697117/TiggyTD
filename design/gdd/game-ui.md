# 游戏 UI 系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27

---

## Overview（概述）

游戏UI系统管理战斗中所有的界面元素：顶部状态栏（基地HP、资源、波次）、底部建造面板（动物塔选择）、技能按钮栏（英雄技能）、敌人血条、状态效果图标、胜负面板。UI层不包含游戏逻辑，只读取游戏状态并渲染，以及将玩家输入（点击按钮）转发给对应系统。

---

## Player Fantasy（玩家幻想）

UI是游戏信息的「仪表盘」。好的UI让玩家在紧张波次中一眼读懂战场状态——基地HP还剩几格、金币够不够放下一只塔、英雄技能好了没。信息越清晰，玩家决策越快，「来得及」和「来不及」的紧张感才能准确传递。MVP的UI要做到「功能完整但不花哨」——所有关键信息可见，所有操作可达，视觉不干扰战场判读。

## Detailed Design（详细设计）

### UI布局总览

```
┌─────────────────────────────────────────┐
│  [基地HP: ♥♥♥♥♥♥♥♥♥♥]  [金币: 150]  [波次: 3/10]  [▶下一波]  │  ← 顶部状态栏
├─────────────────────────────────────────┤
│                                         │
│              游戏地图区域               │  ← 地图渲染区
│           (1280 × 832 px)               │
│                                         │
├─────────────────────────────────────────┤
│  [Q:百兽怒吼 30s]  [W:自然之召 45s]  [E:纪元审判 120s]  │  ← 技能栏（底部左）
│  [猎豹 50g][乌龟 40g][鬣狗 60g][狼群 45g][大象 80g]...  │  ← 建造面板（底部右）
└─────────────────────────────────────────┘
整体分辨率：1280×960
顶部状态栏：1280×64px
地图区域：1280×768px（含底部UI后实际地图为1280×768，底部UI覆盖128px）
底部UI栏：1280×128px
```

### 顶部状态栏

```
基地HP：
  - 显示方式：10个心形图标，有HP=红色实心，无HP=灰色空心
  - 扣血时：对应心形变灰，闪烁红色0.5秒
  - 位置：左侧

金币显示：
  - 格式：硬币图标 + 数字
  - 实时更新（每次收入/支出立即刷新）
  - 位置：中左

波次显示：
  - 格式：「波次 X / 10」
  - 位置：中右

下一波按钮：
  - 仅在 build 阶段显示（波次间隙）
  - 战斗中隐藏
  - 点击 → 调用 wave_system.start_next_wave()
  - 位置：右侧
```

### 底部技能栏

```
每个技能按钮包含：
  - 技能图标（64×64px）
  - 技能名称（文字标签）
  - 冷却倒计时（CD中：灰色遮罩 + 倒计时数字）
  - 快捷键提示（Q/W/E）

状态：
  READY：正常显示，可点击
  COOLDOWN：灰色遮罩，显示剩余秒数，不可点击
  CASTING：高亮边框（施法中）
```

### 底部建造面板

```
每个动物塔按钮包含：
  - 动物图标（48×48px）
  - 动物名称
  - 费用（金币图标 + 数字）
  - 状态：
    AVAILABLE：正常显示，可点击
    INSUFFICIENT_GOLD：金币数字变红，不可点击（金币不足）
    SELECTED：高亮边框（已选中，点击地图放置）

MVP显示动物：猎豹、乌龟、鬣狗、狼群、大象（前5种）
（超出显示范围的动物通过左右箭头翻页，MVP暂不实现翻页，仅显示5种）
```

### 敌人血条

```
每只敌人头顶显示：
  - 血条宽度：32px（按HP比例填充）
  - 满血：绿色；半血以下：黄色；1/4以下：红色
  - 中毒状态：血条上方显示绿色毒液图标
  - 减速状态：血条上方显示蓝色雪花图标
  - 石化状态：血条上方显示灰色石头图标
  - 恐惧状态：血条上方显示黄色骷髅图标
  - 血条始终朝向摄像机（Billboard）
```

### 格子交互高亮

```
鼠标悬停在 BUILD 格：浅绿色半透明高亮
鼠标悬停在 PATH/BLOCKED 格：红色半透明高亮（不可放置）
已选中塔类型时，鼠标移动到格子上显示塔的放置预览（半透明动物精灵）
```

### 胜负面板

```
胜利面板（居中全屏半透明遮罩）：
  - 标题：「胜利！」（金色大字）
  - 统计：基地HP剩余 / 击杀总数 / 通关波次
  - 按钮：「再来一局」/ 「返回主菜单」

失败面板（居中全屏半透明遮罩）：
  - 标题：「失败」（红色大字）
  - 统计：撑到第X波 / 击杀总数
  - 按钮：「再来一局」/ 「返回主菜单」
```

## Formulas（公式）

**血条宽度计算**
```
bar_width = 32 × (current_hp / max_hp)
最小显示宽度：1px（濒死状态仍可见）
```

**技能冷却遮罩填充比例**
```
mask_fill = cooldown_remaining / max_cooldown
从上往下扫描（类时钟冷却遮罩）
```

**金币不足判定**
```
if player.gold < tower.cost:
  button.state = INSUFFICIENT_GOLD
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 敌人同时有多个状态效果图标 | 图标横向排列，最多显示4个，超出以「+N」表示 |
| 金币数字超过4位（≥10000） | 显示「9999+」，MVP阶段金币不会超过此值 |
| 建造面板在波次进行中被点击 | 允许，玩家可在战斗中放塔（若有足够金币） |
| 技能按钮在敌人AI冻结（胜负）后被点击 | game_state != PLAYING时所有按钮禁用 |
| 屏幕分辨率不是1280×960 | MVP固定分辨率，不做自适应缩放 |
| 多个敌人血条重叠 | 按敌人Y坐标排序渲染，Y值大的（靠近底部）在上层显示 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 顶部状态栏高度 | 64px | 48~80px | 越高信息显示越大，但地图可视区域越小 |
| 底部UI栏高度 | 128px | 96~160px | 越高按钮越大易点击，但地图更小 |
| 敌人血条宽度 | 32px | 24~48px | 越宽血量变化越直观 |
| 血条颜色阈值（黄色） | 50% HP | 30~60% | 越低玩家更晚感知到敌人濒死 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 顶部状态栏实时显示基地HP、金币、波次 | 击杀敌人获得金币，确认UI金币数字立即更新 |
| AC-2 | 扣血时对应心形图标变灰并闪烁红色 | 让敌人到达终点，确认一个心形变灰且闪烁0.5秒 |
| AC-3 | 技能CD中按钮显示灰色遮罩和倒计时数字 | 触发百兽怒吼后，确认Q按钮灰色遮罩从上往下扫，显示「30」倒计数 |
| AC-4 | 金币不足时建造面板按钮费用数字变红 | 金币=0时，确认所有塔按钮费用数字变红且无法点击 |
| AC-5 | 敌人血条颜色随HP变化（绿→黄→红） | 让敌人受伤至50%以下，确认血条变黄；25%以下变红 |
| AC-6 | 状态效果图标正确显示在敌人血条旁 | 对敌人施加减速，确认血条旁出现蓝色雪花图标 |
| AC-7 | BUILD格悬停显示绿色高亮，PATH格显示红色高亮 | 鼠标移到路径格，确认红色高亮；移到草地格，确认绿色高亮 |
| AC-8 | 胜利/失败面板正确显示统计数据 | 通关后确认胜利面板显示正确的剩余HP和击杀数 |
