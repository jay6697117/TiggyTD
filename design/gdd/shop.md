# 商店系统

> **状态**: 已完成
> **作者**: 开发者 + Claude
> **最后更新**: 2026-03-27

---

## Overview（概述）

商店系统是 Roguelite 循环的关键节点：每隔若干波次（第3、6、9波后）触发商店阶段，玩家可花费金币购买道具、刷新商品列表或跳过。商店提供4个道具槽位，每次从道具数据库按稀有度权重随机生成。商店是玩家Build决策的核心时刻——买什么、攒钱等下次、还是刷新找特定道具，这些选择构成每局独特的策略张力。

---

## Player Fantasy（玩家幻想）

商店阶段是每局的「喘息+决策」节点。刚刚经历一波艰难战斗，商店界面打开，看到恰好有一件想要的合成材料——这种「运气来了」的惊喜感是 Roguelite 重复游玩的核心钩子。花30金币刷新，刷出一件传说级道具……还是忍住把钱攒着买更多塔？这个张力让每次商店阶段都充满戏剧性。

## Detailed Design（详细设计）

### 商店触发时机

```
商店在以下波次结束后触发：
  第3波结束 → 商店阶段
  第6波结束 → 商店阶段
  第9波结束 → 商店阶段（最后一次，下一波是第10波Boss）
共3次商店机会
```

### 商店结构

```
商店槽位：4个道具槽
每个槽位显示：道具图标、名称、稀有度颜色、价格
操作按钮：
  [购买] - 每件道具独立购买
  [刷新] - 消耗30金币，重新随机生成4件道具
  [关闭] - 结束商店阶段，开始下一波build准备
```

### 商品生成

```
generate_shop_items(wave_number):
  // 根据波次调整稀有度权重
  if wave_number == 3:
    weights = { common: 70%, uncommon: 25%, rare: 5% }
  elif wave_number == 6:
    weights = { common: 50%, uncommon: 35%, rare: 15% }
  elif wave_number == 9:
    weights = { common: 30%, uncommon: 40%, rare: 30% }

  for i in range(4):
    rarity = weighted_random(weights)  // 随机性系统
    pool = ItemDatabase.get_by_rarity(rarity)
    item = random.choice(pool)
    slots[i] = { item: item, price: item.shop_price }
```

### 购买流程

```
buy_item(slot_index):
  item = slots[slot_index].item
  price = slots[slot_index].price
  if player.gold < price:
    show_error("金币不足")
    return
  if inventory.is_full():
    show_error("背包已满")
    return
  player.gold -= price
  inventory.add(item)
  slots[slot_index] = null  // 该槽位变为空（已售出）
```

### 刷新流程

```
refresh_shop():
  if player.gold < 30:
    show_error("金币不足（需要30金币刷新）")
    return
  player.gold -= 30
  重新执行 generate_shop_items(current_wave)  // 所有4个槽位重新随机
```

### 商店阶段时间

```
商店阶段无时间限制（玩家手动关闭）
关闭商店后进入build准备阶段（可放置/调整塔）
build准备阶段也无时间限制（玩家手动开始下一波）
```

## Formulas（公式）

**道具价格**
```
common 道具：20~40金币
uncommon 道具：50~80金币
rare 道具：100~150金币
（具体价格在道具数据库中定义）
```

**刷新期望价值**
```
刷新费用：30金币
第6波刷新后期望稀有度提升：uncommon从25%→35%，rare从5%→15%
若当前4件均为common，刷新期望获得uncommon/rare概率=50%（第6波）
刷新是否值得取决于当前商品质量和玩家Build目标
```

**商店总金币压力**
```
三次商店：每次平均购买1~2件道具
平均花费：50~100金币/次商店
共需150~300金币用于道具
加上塔放置费用，玩家需要合理规划每波金币收益
```

## Edge Cases（边界情况）

| 情况 | 处理方式 |
|------|----------|
| 金币不足30时尝试刷新 | 拒绝刷新，提示「刷新需要30金币」 |
| 背包满时尝试购买 | 拒绝购买，提示「背包已满，请先装备道具」 |
| 4件道具全买完后的商店外观 | 4个槽位全变空，可继续刷新（花费30金币重新生成4件） |
| 刷新后恰好与之前相同 | 允许发生（随机性），不做去重处理 |
| 玩家不打开商店直接进入下一波 | 商店阶段超时不会自动关闭（无超时），玩家必须手动关闭 |
| 第9波商店购买后金币为0 | 正常，第10波（Boss波）通常不需要再建塔 |

## Tuning Knobs（调节旋钮）

| 旋钮 | 当前值 | 安全范围 | 影响的游戏体验 |
|------|--------|---------|---------------|
| 商店触发波次 | 3/6/9波 | 可改为2/5/8等 | 越早触发Build成型越快，节奏感不同 |
| 商店槽位数量 | 4个 | 3~6个 | 越多选择越丰富，决策负担也越重 |
| 刷新费用 | 30金币 | 20~50金币 | 越便宜玩家越倾向于刷新追特定道具 |
| 第9波rare出现率 | 30% | 20~50% | 越高最终Build越强，通关难度降低 |
| common道具价格 | 20~40金币 | 15~60金币 | 影响早期Build速度 |

## Acceptance Criteria（验收标准）

| # | 标准 | 验证方法 |
|---|------|----------|
| AC-1 | 第3波结束后自动弹出商店界面 | 完成第3波，确认商店UI自动弹出，显示4件道具 |
| AC-2 | 购买道具扣除对应金币，道具进入背包 | 购买40金币的道具，确认金币减少40，背包出现该道具 |
| AC-3 | 已购买槽位变为空，其他槽位不变 | 购买第2个槽位的道具，确认只有第2槽变空 |
| AC-4 | 刷新消耗30金币并重新生成4件道具 | 点击刷新，确认金币-30，4个槽位全部更新为新道具 |
| AC-5 | 金币不足时购买和刷新均被拒绝 | 金币=10时尝试购买20金币道具，确认提示「金币不足」 |
| AC-6 | 第9波商店稀有度权重高于第3波 | 统计多次测试中第9波商店rare出现次数，应显著高于第3波 |
| AC-7 | 背包满时拒绝购买并提示 | 背包6格全满时点击购买，确认提示「背包已满」 |
